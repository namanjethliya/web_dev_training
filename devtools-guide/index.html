Opening DevTools

Three ways to open DevTools in any browser. You'll use these every day, so memorize the keyboard shortcuts.

Right-click Method

Right-click on any element on the page → click "Inspect".

DevTools opens with that element already selected in the Elements panel. Fastest way to inspect a specific element.

Docking Position

DevTools can dock to different sides of your screen. Click the ⋮ (three-dot menu) in the top-right of the DevTools panel to change docking.

Dock to Bottom

Good for wide monitors. You see page and DevTools at the same time.

Dock to Right (Recommended)

Best for most use cases. Page on the left, DevTools on the right.

Undock (Separate Window)

Useful with two monitors — DevTools on one screen, page on the other.

02

The Panels

DevTools is split into several tabs (panels). As a beginner, you'll live in just three of them. The rest you'll use later.

The Big Three (Learn These First)

Elements

Shows the HTML structure of the page. Click any element to select it. See and edit its CSS on the right. This is where you'll spend 80% of your time.

Console

Shows JavaScript errors, warnings, and log messages. You can also type and run JavaScript directly here.

Sources

See your HTML, CSS, and JavaScript files as the browser has loaded them. Useful for reading your actual code.

Other Panels (For Later)

PanelWhat it doesWhen you'll use it
NetworkShows every file the browser loads (images, CSS, JS, API calls)When images aren't loading or pages are slow
PerformanceRecords and profiles page loading and rendering speedAdvanced — when optimizing slow pages
ApplicationInspect cookies, localStorage, sessionStorageWhen building forms or auth flows
LighthouseAudits your page for performance, SEO, accessibilityBefore launching a site
Tip: If you don't see a panel, click the >> arrow at the end of the panel tab bar to reveal hidden panels.

Selecting Elements

Method 1 — Click in HTML tree

Click any line in the Elements panel to select that element. The element highlights on the page.

Method 2 — Inspector Cursor

Click the cursor icon (top-left of DevTools) then click any element on the page. DevTools jumps to it in the tree.

Mac: Cmd+Shift+C

Method 3 — Right-click

Right-click any element on the page → "Inspect". Fastest for targeting a specific element.

Editing HTML Live

All edits are temporary — refreshing the page resets everything. This is for experimenting, not saving.

Edit Text Content

Double-click the text inside a tag to edit it inline. Press Enter to confirm.

Edit as HTML

Right-click an element in the tree → "Edit as HTML". A text area opens where you can rewrite the entire element.

Edit Attributes

Double-click any attribute (like class="...") to edit it. Great for testing class changes.

Add/Remove Class

In the Styles panel, click .cls to toggle classes on and off without editing HTML.

Remember: Changes in DevTools are NOT saved to your files. They only exist in the browser's memory. Refresh the page and they're gone.

1 Source file and line number — click to jump to that line in your CSS file
2 Strikethrough rules — being overridden by a more specific rule above it
3 Inherited rules — styles coming from a parent element like body
4 Editable values — click any value to change it live. Great for experimenting.

Editing CSS Live

Change a value

Click any CSS value in the Styles panel and type a new one. The page updates instantly.

Toggle a property

Hover over a rule — a checkbox appears. Uncheck it to disable that property. Check it again to re-enable.

Add a new property

Click the empty space below any rule's last property. Type a new property name and value.

Nudge number values

Click a number value (like 24px) and press ↑ / ↓ to increment/decrement. Hold Shift to step by 10.

The Computed Tab

The Computed tab (next to Styles) shows the final resolved value of every CSS property — after all inheritance, specificity, and shorthand expansion is resolved.

Use it when…

A style isn't working and you can't figure out why. The Computed tab shows exactly what value the browser is actually using.

Trace back to source

Click the arrow next to any computed value to jump to the exact CSS rule that set it. Traces the source even through inheritance.

Force Element States

Click the :hov button in the Styles panel to force an element into a state like :hover, :focus, or :active — without having to actually hover or click it.

This is essential for debugging hover and focus styles — you can inspect the CSS that only appears on mouse hover.

05

The Box Model Visualizer

The Computed tab has a visual box model diagram showing the exact margin, border, padding, and content size of the selected element.

Reading the Diagram

margin
border
padding
320 × 48
16 24 16 24

Each layer shows the pixel value for top / right / bottom / left. Click any value to edit it.

What Each Layer Means

Margin (outermost)

Space outside the border. Pushes other elements away. Never has a background color.

Border

The visible edge of the element. Has width, style, and color.

Padding

Space between the border and the content. Takes the element's background color.

Content (innermost)

The actual text, image, or child elements. Controlled by width and height.

Why This is Useful

Debugging spacing

Something has unexpected space around it? Select the element and check the box model. You'll instantly see if it's margin, padding, or border causing it.

Finding exact sizes

The content area shows the actual rendered pixel width and height — useful when you need precise sizing.

Quick trick: In the Elements panel, hover over any element in the HTML tree — the browser highlights it on the page with the box model layers colored in (blue = content, green = padding, orange = margin).

What You Can Do

Pick a device

Choose a preset like "iPhone 14" or "iPad Air" from the dropdown. Sets exact screen size and pixel ratio.

Drag to resize

In "Responsive" mode, drag the blue handles to set any width. Watch your media queries kick in as you resize.

Type exact size

Type a width and height in the number inputs. Useful for testing specific breakpoints (e.g. 768px).

Throttle network

Simulate slow mobile internet. See how fast (or slow) your page loads on a phone connection.

Important: Device mode simulates screen size but does NOT simulate a real phone's CPU, memory, or touch input. Always test on a real device before launching.

The Console

The Console shows messages from your code — errors, warnings, and anything you print with console.log(). It's also a JavaScript sandbox where you can run code directly.

Message Types

Console ⊘ All Errors Warnings Info Verbose
✕ Cannot read properties of null (reading 'style') main.js:14
⚠ <img> element is missing required alt attribute index.html:38
ℹ Page loaded in 1.24s performance.js:8
User clicked the button app.js:22
> |

Reading Error Messages

Errors (red)

Something broke. The code stopped working. Always fix these first — they're blocking something from running.

Click the file link (main.js:14) to jump to exactly where it happened.

Warnings (yellow)

Something might be wrong — missing alt on an image, deprecated syntax, etc. The page still works but you should fix these.

Log (white)

Normal messages from console.log(). Usually put there by you or a library for debugging.

console.log() — Your Best Friend

Add console.log() anywhere in your JavaScript to print a value. It shows up in the Console tab.

// Print a simple value
console.log("Hello from JS");

// Print a variable
let name = "Naman";
console.log(name);          // → Naman

// Print an object
let user = { name: "Naman", age: 25 };
console.log(user);          // → {name: "Naman", age: 25}

// Label your output
console.log("User object:", user);

// Other console methods
console.warn("This is a warning");
console.error("This is an error");
console.table([{ a: 1 }, { a: 2 }]);  // Pretty table output

The Console as a Scratch Pad

Click on the prompt line (>) at the bottom of the Console and type any JavaScript. Press Enter to run it.

// Select an element and change it
document.querySelector("h1").style.color = "red";

// Check a CSS variable value
getComputedStyle(document.body).getPropertyValue("--accent");

// Find all images on the page
document.querySelectorAll("img").length;

// Check what's in localStorage
localStorage.getItem("theme");
$0 shortcut: In the console, type $0 to reference the element currently selected in the Elements panel. So $0.style.color = "red" changes the selected element's color instantly.

Pro Tips & Shortcuts

These are the shortcuts and tricks that separate beginners from developers who move fast. Learn one at a time.

Selector Tricks

Copy CSS Selector

Right-click an element in the Elements panel → Copy → Copy selector. Gets you a CSS selector you can paste into your stylesheet.

Copy Element

Right-click → Copy → Copy outerHTML. Copies the full HTML of the element. Useful for grabbing elements from live sites.

Scroll into View

Right-click an element in the tree → Scroll into view. The page scrolls to show that element — great for finding off-screen elements.

CSS Tricks

Color Picker

Click the colored square next to any color value in the Styles panel. A color picker opens — choose any color visually.

Search CSS

Press Ctrl+F inside the Styles panel to search across all CSS rules for the selected element.

See all CSS for a selector

In Sources, press Ctrl+F and search your CSS file. Or use the Coverage panel to see which CSS is unused.

Debugging Workflow

When something doesn't look right, follow this process:

1
Inspect the element

Right-click → Inspect. Select the element that's broken.

2
Check the Styles panel

Look for strikethrough rules. Find the rule that's winning the specificity battle.

3
Check the Computed tab

See the actual value the browser is using. Click the arrow to trace it back to the source rule.

4
Check the Box Model

If spacing is wrong, look at the box model diagram. Is extra space coming from margin, padding, or border?

5
Edit live to confirm the fix

Try your fix in DevTools first. Once it works, copy it to your actual file.

Golden rule: Never guess and refresh. Use DevTools to experiment live, confirm the fix works, then transfer to your code. This saves hours.
