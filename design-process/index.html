Read the Design

Now put on your developer glasses. Stop looking at colors and fonts — start looking for big rectangular blocks. Every web page is just sections stacked on top of each other, like layers in a sandwich.

💡
The key idea: A web page is boxes inside boxes. Big boxes contain smaller boxes. Your first job is always to find the biggest boxes.

What Each Section Contains

1
<header>

The top bar. It appears on every single page of the website — logo on the left, navigation links on the right.

Contains: logo, <nav> with links
2
<section id="hero">

The first big impression. A full-width section with a large headline, a short description, and a button. Always the first section inside <main>.

Contains: <h1>, <p>, <button>
3
<section id="menu">

Three repeated cards in a row — same layout, different content. Whenever you see repeated items like this, you wrap them in a container and repeat the card structure.

Contains: <h2>, 3× <div class="card">
4
<section id="about">

Two columns side by side — text on the left, photo on the right. This is a classic two-column layout. You use a wrapper div with two children inside it.

Contains: text <div> + <img>
5
<footer>

The bottom of every page. Copyright notice, social links, extra navigation. Always use a semantic <footer> tag — never a plain <div>.

Contains: logo, links, <small> copyright
03

How to Decide Your Sections

Here is the exact mental process that developers use every time they look at a new design. Follow these 5 steps in order — every single time.

Step 1
Look at the whole page — find the big horizontal bands

Don't zoom into the details yet. Squint your eyes. What are the big bands of content stacked from top to bottom?

Rule Each distinct horizontal band = one section of your HTML
↓
Step 2
Find the 3 outer parts that every page has

Almost every website in the world is built with the same 3 outer containers. Identify them first.

<header> top bar — logo, nav
<main> all unique page content
<footer> bottom bar — links, copyright
↓
Step 3
Look inside <main> — count the sections

Everything between the header and footer lives in <main>. Count how many distinct groups of content there are.

Question to ask "Does this group have a different topic or purpose from the group above it?" If yes → it's a new <section>.
↓
Step 4
Give each section a clear, meaningful name (id)

The id describes what the section is about — not what it looks like, not its position on the page.

✓ Good names
#hero #about #menu #contact #services #gallery
✗ Bad names
#section1 #div-top #big-blue #box2 #top-part #middle
↓
Step 5
Look inside each section — match the content to HTML tags

Now zoom in. Look at what type of content is inside each section and ask: "Which HTML tag is made for this?"

What you see in the design
HTML tag to use
The biggest, most important headline
→
<h1> (only one per page)
Section headings
→
<h2>, <h3>
A paragraph of text
→
<p>
A clickable button
→
<button> or <a>
A photo or image
→
<img src="..." alt="...">
Navigation links (top bar)
→
<nav> with <a> inside
Repeated cards (3 in a row)
→
wrapper <div> + 3× <div class="card">
Two columns side by side
→
wrapper <div> with 2 children
04

Write the HTML

Now we turn the BrewCo design into code. The rule is simple: always write the outer structure first, then fill in the details inside each section. Big boxes first, small boxes second.

Step 1 — Start with the outer skeleton (same for every page)
brewco.html
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8" />
  <meta name="viewport" content="width=device-width, initial-scale=1.0" />
  <title>BrewCo — The Best Coffee in Town</title>
  <link rel="stylesheet" href="styles.css" />
</head>
<body>

  <header> ... </header>   <!-- Section 1: top bar -->

  <main>                    <!-- Sections 2, 3, 4: all content -->
    ...
  </main>

  <footer> ... </footer>   <!-- Section 5: bottom bar -->

</body>
</html>
Step 2 — Fill in the <header>
Section 1 — header
<header>

  <!-- Logo: clicking it takes you back to the home page -->
  <a href="index.html" class="logo">☕ BrewCo</a>

  <!-- Nav: links to sections on this same page using #id -->
  <nav>
    <a href="#menu">Menu</a>
    <a href="#about">About</a>
    <a href="#visit">Visit</a>
    <a href="#order">Order</a>
  </nav>

</header>
Step 3 — Fill in <main> with all sections
Sections 2, 3 and 4 — inside <main>
<main>

  <!-- ──────────────────────────────────────────────
       SECTION 2 · HERO
       First thing the visitor sees. Big headline + button.
  ────────────────────────────────────────────── -->
  <section id="hero">
    <p>EST. 2018 · MUMBAI</p>
    <h1>The Best Coffee in Town</h1>
    <p>Fresh beans. Warm vibes. Every single day.</p>
    <a href="#menu">See Our Menu →</a>
  </section>

  <!-- ──────────────────────────────────────────────
       SECTION 3 · MENU
       3 repeated cards — same structure, different content.
       Pattern: wrapper div + repeat the card inside it.
  ────────────────────────────────────────────── -->
  <section id="menu">
    <h2>Our Favourites</h2>

    <div class="cards-container">  <!-- wrapper holds all 3 cards -->

      <div class="card">           <!-- card 1 -->
        <img src="espresso.jpg" alt="A shot of espresso" />
        <h3>Espresso</h3>
        <p>₹120</p>
      </div>

      <div class="card">           <!-- card 2 -->
        <img src="latte.jpg" alt="A creamy latte" />
        <h3>Latte</h3>
        <p>₹150</p>
      </div>

      <div class="card">           <!-- card 3 -->
        <img src="coldbrew.jpg" alt="Cold brew coffee" />
        <h3>Cold Brew</h3>
        <p>₹180</p>
      </div>

    </div>
  </section>

  <!-- ──────────────────────────────────────────────
       SECTION 4 · ABOUT
       Two columns: text left, image right.
       Pattern: wrapper div with 2 children inside.
  ────────────────────────────────────────────── -->
  <section id="about">

    <div class="about-text">       <!-- left column -->
      <p>Our Story</p>
      <h2>We started with one cup, one dream.</h2>
      <p>BrewCo was born in a small Mumbai apartment. We source
         our beans directly from farms in Coorg and Chikmagalur.</p>
      <a href="about.html">Read More →</a>
    </div>

    <div class="about-image">      <!-- right column -->
      <img src="farm.jpg" alt="Coffee farm in Coorg" />
    </div>

  </section>

</main>
Step 4 — Fill in the <footer>
Section 5 — footer
<footer>
  <a href="index.html" class="logo">☕ BrewCo</a>

  <nav>
    <a href="https://instagram.com/brewco" target="_blank">Instagram</a>
    <a href="https://twitter.com/brewco"   target="_blank">Twitter</a>
    <a href="mailto:hello@brewco.com">Email</a>
  </nav>

  <small>&copy; 2024 BrewCo. All rights reserved.</small>
</footer>
The Complete Structure at a Glance
<body>
<header> — logo + <nav>
<main>
<section id="hero"> — <h1>, <p>, <a>
<section id="menu"> — <h2>, 3× .card
<section id="about"> — text col + img col
</main>
<footer> — logo + links + <small>
</body>
Remember: CSS is what makes it look like the design. Right now you are only building the structure. An unstyled HTML file is completely normal and expected at this stage.

Ask Yourself These Questions First

Q1
How many big horizontal sections can you count on this page?
Hint: Look from top to bottom. Count the distinct bands of content.
Q2
What are the 3 outer elements every page must have?
Hint: You learned these in Step 2 of the process.
Q3
What id would you give the gallery section? What about the contact section?
Hint: Name it after what it contains, not how it looks.
Q4
The about section has a photo on the LEFT and text on the RIGHT — the opposite of BrewCo. Does this change the HTML or only the CSS?
Hint: HTML is structure. CSS controls left/right positioning.
Q5
What HTML tags would you use for the Name, Email, and Message fields in the contact form?
Hint: Think about <input>, <textarea>, and <form>.
