01

What is JavaScript?

HTML gives a page its content. CSS gives it its look. JavaScript gives it its behaviour — the ability to respond, change, and interact in real time.

The Three Languages of the Web

🏗️
HTML
Structure — the nouns.
Headings, paragraphs, images, buttons.
🎨
CSS
Style — the adjectives.
Colours, fonts, layout, spacing.
⚡
JavaScript
Behaviour — the verbs.
Click, submit, fetch, animate, update.

What JavaScript Can Do

React to user actions

Run code when a user clicks a button, submits a form, or moves their mouse.

Change the page

Add, remove, or modify any HTML element or CSS style after the page loads.

Talk to servers

Fetch data from APIs without refreshing the page — the power behind Gmail and Twitter.

Store data locally

Save preferences, form data, or session state in the browser's localStorage.

How the Browser Runs JavaScript

⚙️

When you open a web page, the browser reads your HTML top-to-bottom. When it hits a <script> tag, it hands the code to the JavaScript engine — V8 in Chrome, SpiderMonkey in Firefox. The engine compiles your code to machine instructions and runs it. All of this happens in milliseconds.

JavaScript Runs in Two Places

In the Browser (Client-Side)

What you're learning now. JS runs on the user's machine. Can read and change the page. Cannot access the filesystem or databases directly.

On a Server (Node.js)

JS can also run outside the browser on a server. Same language, different environment. Covered in Module 8 of this course.

Right now: Open DevTools (F12 or Cmd+Option+J) and click the Console tab. Type 1 + 1 and press Enter. You just ran JavaScript. Everything in this guide you can also try there.

Your First JavaScript

Three ways to run JavaScript — and why console.log() is the first function every developer learns.

Where to Write JavaScript

1. Inline <script> tag

Place a <script> tag at the bottom of your HTML body. JS runs when the browser reaches it.

2. External .js file

Link a separate file: <script src="app.js"></script>. Recommended for all real projects.

3. Browser Console

Type directly in DevTools Console. Great for experimenting — nothing is saved.

The <script> Tag

index.html
<!DOCTYPE html>
<html lang="en">
<head>
  <title>My Page</title>
</head>
<body>
  <h1>Hello</h1>

  <!-- Put script at the bottom of body so HTML loads first -->
  <script src="app.js"></script>
</body>
</html>

console.log() — Your Most Important Tool

console.log() prints a value to the browser console. Use it to check what a variable holds, confirm that code ran, or debug a problem.

app.js
// Print a string
console.log("Hello, world!");

// Print a number
console.log(42);

// Print the result of an expression
console.log(10 + 5);

// Print multiple values at once
console.log("Result:", 10 + 5);

// Other console methods
console.warn("This is a warning");
console.error("This is an error");

Comments

app.js
// Single-line comment — the browser ignores this line

/*
  Multi-line comment.
  Everything between the markers is ignored.
*/

console.log("This runs"); // inline comment — ignored from here to end of line

03

Variables

Variables are named containers for storing values. You create them once and refer to them by name throughout your code.

var, let, and const

JavaScript has three keywords for declaring variables. In modern JS (ES6+), use const by default and let when you need to reassign. Avoid var.

KeywordReassignable?Redeclarable?ScopeUse when…
const ✗ No ✗ No Block Default — use for everything that won't be reassigned
let ✓ Yes ✗ No Block When the value needs to change (counters, loops, toggles)
var ✓ Yes ✓ Yes Function Avoid — legacy syntax with confusing hoisting behaviour

Declaring and Using Variables

app.js
// const — value cannot be reassigned
const name = "Naman";
const pi   = 3.14159;

// let — value can change
let score = 0;
score = 10;      // reassign — OK
score += 5;      // score is now 15

// Naming rules — use camelCase
let myAge = 25;            // ✓ camelCase
let _privateValue = 42;    // ✓ underscore prefix
// let 2fast = true;       // ✗ cannot start with a number
// let my-name = "x";      // ✗ hyphens not allowed

// const does NOT mean "deeply immutable"
const user = { name: "Naman" };
user.name = "Raj";  // ✓ — changing a property is OK
// user = {};       // ✗ — reassigning the variable is not OK

Try It — Variables

Try it
Rule of thumb: Start with const. If you get an error because you tried to reassign it, switch it to let. This forces you to think about whether a value really needs to change.

04

Data Types

Every value in JavaScript has a type. Understanding types is essential — many bugs come from values being a different type than expected.

The 7 Primitive Types

TypeExampletypeof resultUse for
number 42, 3.14, -7 "number" Any numeric value — integers and decimals
string "hello", 'JS', `template` "string" Text — wrap in single, double, or backtick quotes
boolean true, false "boolean" Yes/no, on/off, pass/fail flags
undefined undefined "undefined" Variable declared but not yet assigned a value
null null "object" ⚠️ Intentional absence of a value — you set this explicitly
symbol Symbol("id") "symbol" Unique identifiers (advanced — rarely needed by beginners)
bigint 9007199254740991n "bigint" Integers larger than Number.MAX_SAFE_INTEGER
⚠️

The famous bug: typeof null === "object" — this is a well-known mistake in JavaScript that was never fixed for backwards compatibility. null is NOT an object. To check for null, always use value === null, not typeof value.

undefined vs null

undefined

A variable that exists but has not been given a value yet. JavaScript sets this automatically.

let x;
console.log(x); // undefined
null

A variable explicitly set to "nothing". You set this on purpose to signal the absence of a value.

let user = null;
// "no user logged in"

Try It — typeof on Every Type

Try it
Quick check: Use typeof value to see what type a value is. If you're not sure what's inside a variable, log its typeof — it's often the fastest way to spot a type bug.

Operators

Operators let you compute values, compare them, and combine conditions. The equality operators (== vs ===) are one of JavaScript's most important distinctions.

Arithmetic Operators

OperatorNameExampleResult
+Addition / Concatenation5 + 38
-Subtraction10 - 46
*Multiplication3 * 412
/Division15 / 43.75
%Remainder (Modulo)10 % 31
**Exponentiation2 ** 8256
++Increment by 1let x = 5; x++x is 6
--Decrement by 1let x = 5; x--x is 4

== vs === — The Most Important Distinction

🔑

== (loose equality) converts types before comparing — it will say "5" == 5 is true. This causes subtle bugs.

=== (strict equality) compares value AND type — "5" === 5 is false. Always use ===.

Comparison & Logical Operators

Comparison

=== strict equal  ·  !== strict not equal
> greater  ·  < less
>= greater/equal  ·  <= less/equal

&& — AND

True only if both sides are true.
true && false → false
true && true → true

|| — OR

True if at least one side is true.
true || false → true
false || false → false

! — NOT

Flips true↔false.
!true → false
!false → true

Ternary Operator

Syntax: condition ? valueIfTrue : valueIfFalse
const age = 20;
const status = age >= 18 ? "adult" : "minor";
console.log(status); // "adult"

// Equivalent if/else:
// if (age >= 18) { status = "adult"; } else { status = "minor"; }

// Nested ternary for multiple grades
const score = 75;
const grade = score >= 90 ? "A"
            : score >= 80 ? "B"
            : score >= 70 ? "C"
            : "F";
console.log(grade); // "C"

Try It — Operators

Try it
Rule: Never use == or !=. Always use === and !==. The loose versions cause hard-to-find bugs because of automatic type conversion.

06

Conditional Statements

Conditionals let your code make decisions. Run different code based on whether a condition is true or false.

if / else if / else

app.js
const temperature = 28;

if (temperature > 35) {
  console.log("It's very hot! Stay indoors.");
} else if (temperature > 25) {
  console.log("It's warm. Good weather.");
} else if (temperature > 15) {
  console.log("It's mild. Carry a jacket.");
} else {
  console.log("It's cold. Bundle up!");
}
// Output: "It's warm. Good weather."

switch

Use switch when comparing one value against many specific options. Cleaner than a long chain of else if.

app.js
const day = "Monday";

switch (day) {
  case "Monday":
    console.log("Start of the week!");
    break;              // ← always include break!
  case "Friday":
    console.log("Weekend is near!");
    break;
  case "Saturday":
  case "Sunday":        // ← two cases, one block
    console.log("It's the weekend!");
    break;
  default:
    console.log("Just another weekday.");
}
// Output: "Start of the week!"

Try It — Conditionals

Try it
Order matters: Conditions in an if/else chain are checked top-to-bottom. The first true condition wins and the rest are skipped. Always put your most specific conditions at the top.
Learn more: if/else — MDN ↗ switch — MDN ↗ Conditionals — Medium ↗
07

Loops

Loops repeat a block of code. Instead of writing the same line 100 times, write it once inside a loop and tell it how many times to run.

for Loop — When you know the count

Syntax: for (initialise; condition; increment) { }
for (let i = 0; i < 5; i++) {
  console.log("Count:", i);
}
// Output: Count: 0 · Count: 1 · Count: 2 · Count: 3 · Count: 4

// Sum of 1 to 10
let sum = 0;
for (let i = 1; i <= 10; i++) {
  sum += i;
}
console.log("Sum:", sum); // Sum: 55

while Loop — When you don't know the count

app.js
let count = 1;

while (count <= 3) {
  console.log("while count:", count);
  count++;   // ← don't forget this — infinite loop if missing!
}
// Output: while count: 1 · while count: 2 · while count: 3

for...of — Loop over an array's values

app.js
const fruits = ["apple", "banana", "mango"];

for (const fruit of fruits) {
  console.log("Fruit:", fruit);
}
// Output: Fruit: apple · Fruit: banana · Fruit: mango

break and continue

break — exit loop early
for (let i = 0; i < 10; i++) {
  if (i === 5) break;
  console.log(i);
}
// Logs: 0 1 2 3 4
continue — skip this iteration
for (let i = 0; i < 6; i++) {
  if (i % 2 === 0) continue;
  console.log(i);
}
// Logs: 1 3 5 (skips evens)

Try It — Loops

Try it
Which loop? Use for when you know the count. Use while when the end condition is dynamic. Use for...of to loop over an array's values. Use for...in to loop over an object's keys (covered in Module 3).

08

Type Coercion

JavaScript automatically converts types in certain situations. This is called implicit coercion — and it causes some of the language's most surprising bugs.

Implicit Coercion — The Surprises

These are all real JavaScript outputs — run them to believe it
console.log("5" + 3);      // "53"  — 3 coerced to string, concatenated
console.log("5" - 3);      // 2     — "5" coerced to number, subtracted
console.log("5" * "2");    // 10    — both coerced to numbers
console.log(true + 1);     // 2     — true coerced to 1
console.log(false + 1);    // 1     — false coerced to 0
console.log(null + 1);     // 1     — null coerced to 0
console.log("" + 1);       // "1"   — 1 coerced to string

Explicit Conversion — The Safe Way

app.js
// To Number
Number("42");       // 42
Number("3.14");     // 3.14
Number("abc");      // NaN — can't convert
Number("");         // 0
Number(true);       // 1
Number(false);      // 0
parseInt("42px");   // 42 — stops at non-numeric character
parseFloat("3.14rem"); // 3.14

// To String
String(42);         // "42"
String(true);       // "true"
(42).toString();    // "42"

// To Boolean
Boolean(0);         // false
Boolean("");        // false
Boolean(null);      // false
Boolean(undefined); // false
Boolean(NaN);       // false
Boolean("hello");   // true
Boolean(42);        // true
Boolean([]);        // true  ← empty array is TRUTHY — surprising!

The 6 Falsy Values — Memorise These

In a boolean context (inside an if condition), these 6 values are treated as false. Everything else is truthy.

0 "" null undefined NaN false

Try It — Coercion Gotchas

Try it
Defence: Use explicit conversion (Number(), String(), Boolean()) instead of relying on implicit coercion. Use === instead of ==. These two habits eliminate most coercion bugs.

Report Card Generator

Reference Solution

Only look at this after making a genuine attempt.

report-card.js — full solution
const subjects = ["Maths", "Science", "English", "History", "Computer"];
const scores   = [78, 85, 90, 62, 74];

// Total + average
let total = 0;
for (const score of scores) {
  total += score;
}
const average = total / scores.length;

// Print each subject
for (let i = 0; i < subjects.length; i++) {
  console.log(subjects[i] + ": " + scores[i]);
}

// Assign grade
let grade;
if      (average >= 90) { grade = "A"; }
else if (average >= 80) { grade = "B"; }
else if (average >= 70) { grade = "C"; }
else if (average >= 60) { grade = "D"; }
else                    { grade = "F"; }

// Summary
const passed = average >= 60;
console.log("---");
console.log("Average: " + average.toFixed(2));
console.log("Grade:   " + grade);
console.log(passed ? "Result: PASS ✓" : "Result: FAIL ✗");
When you're done: Try setting all scores to 95 — you should get A + PASS. Try all 40s — you should get F + FAIL. If both work correctly, your logic is solid.
