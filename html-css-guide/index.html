01

Document Structure

Every HTML file starts with the same skeleton. These are the bones that hold everything together.

<!DOCTYPE html>

Tells the browser "this is an HTML5 document." Always the very first line. Not really a tag — it's a declaration.

<html lang="en">

The root element. Everything lives inside it. The lang attribute tells browsers and screen readers what language the page is in.

<head>

Invisible to users. Contains metadata, title, links to CSS, scripts, fonts. The brain of the document.

<body>

Everything visible on the page goes here — text, images, videos, forms, everything.

The Full Skeleton
<!DOCTYPE html>
<html lang="en">

  <head>
    <meta charset="UTF-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <title>My Page Title</title>
    <link rel="stylesheet" href="styles.css" />
  </head>

  <body>
    <!-- Your content goes here -->
  </body>

</html>

Headings

HTML has 6 heading levels, from biggest to smallest. Use them to create hierarchy — like a newspaper.

h1 — Main Title (use once per page)

h2 — Section Title

h3 — Sub-section

h4 — Sub-sub-section

h5 — Small heading
h6 — Smallest heading
Heading Tags
<h1>Main Title — only one per page</h1>
<h2>Section Title</h2>
<h3>Sub-section</h3>
<h4>Sub-sub-section</h4>
<h5>Smaller heading</h5>
<h6>Smallest heading</h6>
Tip: Headings affect SEO (search engines) and accessibility (screen readers). Always keep them in logical order — don't skip from h1 to h4!

Lists

Three types of lists in HTML — unordered, ordered, and definition lists.

<ul> — Unordered List

Bullet points — order doesn't matter.

  • JavaScript

Links & Anchors

The <a> (anchor) tag is what makes the web a web — it connects pages together.

External Link
<a href="https://google.com" target="_blank" rel="noopener">
  Visit Google
</a>

target="_blank" opens in new tab. rel="noopener" is a security best practice.

Internal Page Link
<a href="about.html">About Page</a>

<a href="./contact.html">Contact</a>
Jump to Section (Anchor)
<!-- The link -->
<a href="#section-id">Go to Section</a>

<!-- The target -->
<section id="section-id">
  ...
</section>
Email & Phone Links
<a href="mailto:you@email.com">
  Email me
</a>

<a href="tel:+911234567890">
  Call us
</a>
Download Link
<a href="file.pdf" download="my-file.pdf">
  Download PDF
</a>

The download attribute forces file download instead of opening.

Image as Link
<a href="https://example.com">
  <img src="logo.png" alt="Logo" />
</a>

Wrap any element in <a> to make it clickable.

06

Images

The <img> tag is self-closing. The alt attribute is mandatory — it describes the image for screen readers and when images fail to load.

Image Examples
<!-- Basic image -->
<img src="photo.jpg" alt="A description of the image" />

<!-- With width and height (prevents layout shift) -->
<img src="photo.jpg" alt="Profile photo" width="300" height="200" />

<!-- Responsive image -->
<img src="photo.jpg" alt="Banner" style="width: 100%; height: auto;" />

<!-- Image from the web -->
<img src="https://example.com/photo.jpg" alt="Remote image" />

<!-- Lazy loading (loads only when scrolled into view) -->
<img src="photo.jpg" alt="Lazy loaded" loading="lazy" />

<!-- Figure with caption -->
<figure>
  <img src="chart.png" alt="Sales chart Q1 2024" />
  <figcaption>Figure 1: Sales growth in Q1 2024</figcaption>
</figure>

Image Attributes

AttributeExamplePurpose
src"photo.jpg"Source — path to image file (required)
alt"A red apple"Alternative text (required, accessibility)
width"300"Width in pixels
height"200"Height in pixels
loading"lazy" / "eager"When to load the image
title"Hover tooltip"Tooltip text

<table>
  <caption>Table Title (optional)</caption>

Semantic HTML5

Semantic tags describe the meaning of content, not just its appearance. They help search engines, screen readers, and developers understand your page.

<header>
Logo, nav, hero
<nav>
Navigation links
<main>
<article>
Blog post, news story
<section>
Chapters, tabs
<aside>
Sidebar, ads, related
<footer>
Copyright, links, contact
<header>

Introductory content — logo, site title, main navigation. Used in <body> or inside <article>.

<nav>

Navigation links. Use for main navigation, breadcrumbs, pagination. Not every link group needs this tag.

<main>

The primary unique content of the page. Only ONE <main> per page. Skip repeated content like nav/footer.

<article>

Self-contained content that can stand alone — blog post, news article, comment, product card. Could be syndicated elsewhere.

<section>

Groups related content together. Should have a heading. Think of it as a chapter — use <div> if just styling.

<aside>

Content tangentially related to the main content — sidebars, pull quotes, advertising, related links.

<footer>

Footer section — author info, copyright, links. Can be inside <article> too for per-article footers.

<figure> & <figcaption>

Self-contained media (image, chart, code) with its caption. The caption explains or references the figure.

<figure>
  <img src="chart.png" alt="Data chart" />
  <figcaption>Fig 1: Sales Q1</figcaption>
</figure>
<time>

Machine-readable date/time. The datetime attribute holds the standard format.

Published: June 17, 2024
<address>

Contact information for the nearest ancestor <article> or <body>.

Block vs Inline Elements

The most important layout concept in HTML. Every element is either block-level or inline.

Block Elements
  • Start on a new line
  • Take up the full width available
  • Can contain block and inline elements
div block
Another div block
div p h1–h6 ul ol li table form header footer section article main nav aside blockquote pre hr
Inline Elements
  • Stay on the same line
  • Take only the space they need
  • Cannot contain block elements

This is span and strong inline.

span a strong em b i u s img code kbd mark small sup sub abbr cite q label button
CSS override: You can change any element's display with CSS — display: block, display: inline, or display: inline-block.

Meta & Head Tags

Everything inside <head> — invisible to users but critical for SEO, sharing, and browser behavior.

Complete <head> Reference
<head>
  <!-- Character encoding (always first) -->
  <meta charset="UTF-8" />

  <!-- Responsive design viewport -->
  <meta name="viewport" content="width=device-width, initial-scale=1.0" />

  <!-- Page title — shown in browser tab and Google results -->
  <title>My Awesome Page | Brand Name</title>

  <!-- SEO meta tags -->
  <meta name="description" content="A short summary (150–160 chars) for Google." />
  <meta name="keywords" content="html, css, web, tutorial" />
  <meta name="author" content="Your Name" />
  <meta name="robots" content="index, follow" />

  <!-- Open Graph — how links look on Facebook, WhatsApp -->
  <meta property="og:title" content="My Awesome Page" />
  <meta property="og:description" content="Page description for social sharing." />
  <meta property="og:image" content="https://example.com/thumbnail.jpg" />
  <meta property="og:url" content="https://example.com/page" />
  <meta property="og:type" content="website" />

  <!-- Twitter Card -->
  <meta name="twitter:card" content="summary_large_image" />
  <meta name="twitter:title" content="My Awesome Page" />
  <meta name="twitter:image" content="https://example.com/thumbnail.jpg" />

  <!-- Favicon (browser tab icon) -->
  <link rel="icon" href="favicon.ico" type="image/x-icon" />
  <link rel="icon" href="icon.png" type="image/png" />
  <link rel="apple-touch-icon" href="apple-icon.png" />

  <!-- External CSS -->
  <link rel="stylesheet" href="styles.css" />

  <!-- Preload important resources -->
  <link rel="preload" href="hero-image.jpg" as="image" />
  <link rel="preconnect" href="https://fonts.googleapis.com" />

  <!-- Theme color for mobile browsers -->
  <meta name="theme-color" content="#e94560" />

  <!-- Redirect after 5 seconds -->
  <!-- <meta http-equiv="refresh" content="5; url=https://example.com" /> -->

  <!-- Internal CSS -->
  <style>
    body { margin: 0; }
  </style>

  <!-- JavaScript (defer = load after HTML parses) -->
  <script src="app.js" defer></script>
</head>
