01

What is the DOM?

The DOM (Document Object Model) is the browser's live, JavaScript-accessible representation of your HTML page. It is a tree of objects — every element, attribute, and piece of text is a node.

The Browser's Mental Model

When your browser loads an HTML file, it reads the markup and builds a tree of objects in memory. JavaScript can read and change this tree at any time — and the page updates instantly to reflect the change.

🌳

HTML is static text. The DOM is a live tree. When JavaScript changes element.textContent, it changes the DOM node — the original HTML file on disk is untouched. Reload the page and it resets.

The Window / Document / Element Hierarchy

window

The global object in the browser. Everything lives on window. window.document, window.location, window.alert(). You can omit "window." — document and alert() work directly.

document

Represents the entire loaded HTML page. It's your entry point into the DOM. Use document.querySelector(), document.createElement(), etc. to interact with the page.

Element

Each HTML tag — <div>, <p>, <button> — is an Element node. Elements have properties like .textContent, .style, .classList, and methods like .addEventListener().

Node Types

Node TypeExampleHow to get it
Element Node <div>, <p>, <button> document.querySelector()
Text Node The text "Hello" inside an <h1> element.firstChild (when it's text)
Document Node The root document object document global
Attribute Node class="btn", id="title" element.getAttribute()
Key insight: The DOM is a live representation. When you run document.querySelector('h1').textContent = 'Goodbye', the <h1> on screen changes immediately — no page reload needed. This is the entire power of JavaScript on the web.

Selecting Elements

Before you can change anything on a page, you need to select the element. JavaScript gives you several methods — each with its own strengths.

The Four Core Selection Methods

MethodReturnsSelector typeLive?
getElementById(id) Single element or null ID string only —
querySelector(sel) First match or null Any CSS selector No
querySelectorAll(sel) Static NodeList (array-like) Any CSS selector No
getElementsByClassName(c) Live HTMLCollection Class name(s) Yes

getElementById — The Fastest, Most Specific

app.js
// IDs must be unique on the page — getElementById is the fastest selector
const title = document.getElementById('title');

console.log(title);           // the element node
console.log(title.tagName);   // "H1"
console.log(title.id);        // "title"

// If no element has that ID, returns null
const missing = document.getElementById('does-not-exist');
console.log(missing);         // null

querySelector — The Modern Standard

querySelector accepts any valid CSS selector. It returns the first matching element, or null if nothing matches. This is the method you'll use most often.

app.js
// Select by tag
const h1 = document.querySelector('h1');

// Select by class
const intro = document.querySelector('.hero-sub');

// Select by ID (same as getElementById, but with # prefix)
const title = document.querySelector('#title');

// Select by attribute
const link = document.querySelector('a[href="#selecting"]');

// Select with combinators — just like CSS
const firstBtn = document.querySelector('.sandbox .sandbox-run');

// Chain: search WITHIN an element
const nav = document.querySelector('nav');
const firstLink = nav.querySelector('a');  // first <a> inside the nav

querySelectorAll — Multiple Elements

querySelectorAll returns a NodeList of all matching elements. It looks like an array and you can loop over it, but it's not a true array — use Array.from() to get array methods like .map().

app.js
// Get ALL paragraphs
const allPs = document.querySelectorAll('p');
console.log(allPs.length); // number of <p> elements

// Loop with forEach — NodeList supports this natively
allPs.forEach(p => {
  console.log(p.textContent.slice(0, 40));
});

// Convert to array for full array methods
const pArray = Array.from(allPs);
const texts = pArray.map(p => p.textContent);

// Select all items with a specific class
const pills = document.querySelectorAll('.pill');
pills.forEach(pill => {
  console.log(pill.textContent); // "11 Chapters", "Live DOM Demos", ...
});

When to Use Which

getElementById

When you have a unique ID and need that one specific element. Marginally faster than querySelector — useful in performance-critical loops.

querySelector ← use this most

The Swiss army knife. Accepts any CSS selector. Consistent, readable, and handles all your single-element needs.

querySelectorAll

When you need to select multiple elements and iterate over them — a list of items, all buttons in a section, etc.

03

Reading & Modifying Content

Once you've selected an element, you can read or change its text, HTML, or both. Three properties handle this — and their differences matter.

textContent vs innerHTML vs innerText

PropertyReadsWritesParses HTML tags?Use when…
textContent All text, including hidden Sets text — escapes HTML No — tags shown as text Reading or setting plain text (safest)
innerHTML HTML markup + text Parses and renders HTML Yes Inserting HTML from a trusted source
innerText Visible text only Sets visible text No When you need layout-aware text (slower)

textContent — Safe and Simple

app.js
const heading = document.querySelector('h1');

// Read the text
console.log(heading.textContent); // "JavaScript DOM & Events"

// Write text — if you write HTML tags, they show as literal text
heading.textContent = 'New Heading';

// Safe: user input can't inject HTML this way
const userInput = '<img src=x onerror=alert(1)>';
heading.textContent = userInput; // shows the literal string — not an image

innerHTML — Powerful but Dangerous

app.js
const card = document.querySelector('.card');

// Read the HTML inside
console.log(card.innerHTML);

// Write HTML — the browser parses and renders it
card.innerHTML = '<strong>Bold text</strong> and <em>italic</em>';

// Completely replace content with a structured block
card.innerHTML = `
  <h3>Updated Card</h3>
  <p>This content was set via innerHTML.</p>
`;
⚠️

XSS Security Warning: Never set innerHTML to a string that contains user input. An attacker can inject malicious scripts: element.innerHTML = userInput where userInput = "<img src=x onerror=alert('hacked')>" executes JavaScript. Always use textContent for user data, or sanitise HTML with a library like DOMPurify.

innerText — Layout-Aware

app.js — textContent vs innerText difference
// Given: <p>Hello <span style="display:none">hidden</span> World</p>
const p = document.querySelector('p');

console.log(p.textContent); // "Hello hidden World" — includes hidden text
console.log(p.innerText);   // "Hello  World" — only visible text

// innerText is slower because it forces a layout calculation
// For most use cases, prefer textContent

04

Modifying Styles & Classes

JavaScript can change an element's appearance in two ways: by directly setting inline styles, or by adding/removing CSS classes. Classes are almost always the better approach.

element.style — Inline Styles

The style property maps to the element's inline style attribute. CSS property names use camelCase in JavaScript: background-color becomes backgroundColor.

app.js
const box = document.querySelector('.card');

// Set a single style property
box.style.color = 'red';
box.style.backgroundColor = '#fff3cd';  // camelCase!
box.style.fontSize = '1.2rem';
box.style.border = '2px solid orange';

// Read a style (only returns inline styles — not stylesheet styles)
console.log(box.style.color); // "red"

// Remove an inline style by setting it to empty string
box.style.color = '';   // browser reverts to stylesheet value

// Set multiple styles at once with cssText
box.style.cssText = 'color: blue; font-size: 1.1rem; font-weight: bold;';
⚠️

element.style only reads inline styles. If a color is set in a stylesheet, element.style.color returns "". To read computed styles (all applied CSS), use window.getComputedStyle(element).color.

classList — The Better Way

Instead of setting individual styles, toggle CSS classes. This keeps styling in CSS where it belongs, and makes your JS cleaner and more maintainable.

MethodEffectExample
classList.add(cls)Add a classel.classList.add('active')
classList.remove(cls)Remove a classel.classList.remove('hidden')
classList.toggle(cls)Add if absent, remove if presentel.classList.toggle('open')
classList.contains(cls)Returns true/falseel.classList.contains('active')
classList.replace(old, new)Swap one class for anotherel.classList.replace('btn-red', 'btn-green')
app.js
const btn = document.querySelector('.sandbox-run');

// Add a class
btn.classList.add('active');
console.log(btn.className); // "sandbox-run active"

// Check if class exists
console.log(btn.classList.contains('active')); // true

// Toggle — great for show/hide, dark mode, menus
btn.classList.toggle('active'); // removes 'active'
btn.classList.toggle('active'); // adds 'active' back

// Remove
btn.classList.remove('active');
console.log(btn.classList.contains('active')); // false

// Add multiple classes at once
btn.classList.add('highlighted', 'large');

Try it — live style changes
Best practice: Prefer classList over element.style. Define your visual states in CSS (.active, .hidden, .error) and toggle those classes in JS. This separates concerns and makes both CSS and JS easier to maintain.

Element Attributes

Attributes like href, src, disabled, and data-* are part of HTML elements. JavaScript lets you read, set, and remove them programmatically.

getAttribute / setAttribute / removeAttribute

app.js
const link = document.querySelector('a');

// Read an attribute
console.log(link.getAttribute('href'));       // e.g. "#selecting"
console.log(link.getAttribute('class'));      // class string

// Set an attribute
link.setAttribute('href', 'https://mdn.io');
link.setAttribute('target', '_blank');

// Check if attribute exists
console.log(link.hasAttribute('disabled'));   // false

// Remove an attribute
link.removeAttribute('target');

// Some attributes are also direct DOM properties (faster to use these)
const img = document.querySelector('img');
// img.src = 'new.png';      // same as setAttribute('src', 'new.png')
// img.alt = 'description';
const btn = document.querySelector('button');
btn.disabled = true;          // same as setAttribute('disabled', '')
btn.disabled = false;         // same as removeAttribute('disabled')

dataset — Custom data-* Attributes

The data-* convention lets you store custom data directly on HTML elements. Access all data-* attributes through the dataset property — camelCase automatically.

HTML + JS — data-* attributes
<!-- In HTML -->
<button
  data-user-id="42"
  data-action="delete"
  data-confirm="Are you sure?"
>Delete</button>
app.js
const btn = document.querySelector('button');

// Read: data-user-id → dataset.userId  (kebab → camelCase)
console.log(btn.dataset.userId);    // "42"
console.log(btn.dataset.action);    // "delete"
console.log(btn.dataset.confirm);   // "Are you sure?"

// Set a new data attribute
btn.dataset.role = 'admin';
// This adds data-role="admin" to the element

// Delete a data attribute
delete btn.dataset.action;

// Useful pattern: tag items with an ID for event handling
// <li data-id="5">Task</li>
// In click handler: const id = event.target.dataset.id;

Creating & Inserting Elements

You can build new HTML elements entirely in JavaScript and insert them anywhere in the DOM — no page reload needed.

createElement + appendChild — The Classic Pattern

app.js
// 1. Create a new element
const newParagraph = document.createElement('p');

// 2. Set its content and attributes
newParagraph.textContent = 'This paragraph was created by JavaScript!';
newParagraph.classList.add('dynamic-text');
newParagraph.style.color = 'teal';

// 3. Insert it into the DOM — appendChild adds as the LAST child
const container = document.querySelector('.main-content');
container.appendChild(newParagraph);

// Build a complete card programmatically
const card = document.createElement('div');
card.className = 'card';

const heading = document.createElement('h3');
heading.textContent = 'New Card';

const para = document.createElement('p');
para.textContent = 'Built entirely in JavaScript.';

card.appendChild(heading);
card.appendChild(para);
container.appendChild(card);

insertAdjacentHTML — Fastest for HTML Strings

insertAdjacentHTML(position, htmlString) parses an HTML string and inserts it at a specific position relative to the element. Four positions:

Position stringWhere it inserts
'beforebegin'Before the element itself (as a sibling before)
'afterbegin'Inside the element, before its first child
'beforeend'Inside the element, after its last child (like appendChild)
'afterend'After the element itself (as a sibling after)
app.js
const list = document.querySelector('ul');

// Add to end (most common)
list.insertAdjacentHTML('beforeend', '<li>New last item</li>');

// Add to beginning
list.insertAdjacentHTML('afterbegin', '<li>New first item</li>');

// Insert after the list
list.insertAdjacentHTML('afterend', '<p>This is after the list.</p>');

insertBefore and cloneNode

app.js
// insertBefore(newNode, referenceNode)
const parent = document.querySelector('.toc-list');
const newItem = document.createElement('li');
newItem.textContent = 'Prepended item';
const firstItem = parent.firstElementChild;
parent.insertBefore(newItem, firstItem); // inserts before first child

// cloneNode — copy an existing element
const originalCard = document.querySelector('.card');
const clone = originalCard.cloneNode(true); // true = deep clone (includes children)
clone.querySelector('.card-tag').textContent = 'Cloned Card';
originalCard.parentNode.appendChild(clone);

Removing Elements

Once you can create elements, you need to know how to remove them too. Three methods cover all scenarios.

The Three Removal Methods

MethodCalled onEffectReturns
element.remove() The element itself Removes element from DOM undefined
parent.removeChild(child) The parent element Removes child from parent The removed child node
element.replaceWith(…) The element to replace Replaces element with new node(s) undefined

element.remove() — The Modern Way

app.js
// The cleanest way — call .remove() directly on the element
const notification = document.querySelector('.notification-banner');
notification.remove();  // gone!

// Common pattern: remove after a delay
setTimeout(() => {
  const alert = document.querySelector('.alert');
  if (alert) alert.remove();
}, 3000); // disappears after 3 seconds

// Remove all items matching a selector
document.querySelectorAll('.temp-item').forEach(el => el.remove());

removeChild() — The Legacy Way

app.js
const list = document.querySelector('ul');
const firstItem = list.firstElementChild;

// Remove first child — you need the parent
const removed = list.removeChild(firstItem);
console.log('Removed:', removed.textContent);

// The removed node is returned — you can re-insert it elsewhere
anotherList.appendChild(removed);

replaceWith() — Remove and Replace in One Step

app.js
const oldHeading = document.querySelector('h2');

// Replace with a new element
const newHeading = document.createElement('h2');
newHeading.textContent = 'Updated Heading';
newHeading.classList.add('updated');
oldHeading.replaceWith(newHeading);

// Replace with an HTML string (use insertAdjacentHTML for this instead)
// replaceWith accepts multiple nodes or strings
oldHeading.replaceWith('Plain text replacement');
oldHeading.replaceWith(node1, node2, 'some text');

Try it — remove page elements
Use .remove() in modern code. removeChild() requires you to have a reference to the parent, which is clunky. element.remove() is supported in all modern browsers and is far cleaner. Only use removeChild() when you need the returned reference to re-use the removed node.

Event Listeners

Events are signals the browser fires when something happens — a click, a keypress, a page load. addEventListener is how you listen for them and run code in response.

addEventListener Syntax

Syntax: element.addEventListener(eventType, handler, options)
// Basic pattern
element.addEventListener('click', function(event) {
  // runs when element is clicked
  console.log('Clicked!', event);
});

// Arrow function (more common in modern code)
element.addEventListener('click', (event) => {
  console.log('Clicked!', event.target);
});

// Named function — reusable and removable
function handleClick(event) {
  console.log('Clicked!');
}
element.addEventListener('click', handleClick);

// The third argument: options object
element.addEventListener('click', handleClick, {
  once: true,    // fires only once, then auto-removes
  capture: true, // use capture phase (default is bubble phase)
  passive: true  // signal that you won't call preventDefault (improves scroll perf)
});

removeEventListener

app.js — removing listeners
// To remove a listener, you MUST pass the same function reference
function handleClick(event) {
  console.log('clicked');
}

btn.addEventListener('click', handleClick);
// ... later ...
btn.removeEventListener('click', handleClick);  // ✓ works

// This does NOT work — anonymous function creates a new reference each time
btn.addEventListener('click', () => console.log('click'));
btn.removeEventListener('click', () => console.log('click')); // ✗ no-op

The Event Object

Every event handler receives an event object as its first argument. It contains everything about the event — what was clicked, what key was pressed, where the mouse was, and more.

Core Event Object Properties

Property / MethodTypeDescription
event.typestringEvent name: "click", "keydown", "input", etc.
event.targetElementThe element that was directly interacted with
event.currentTargetElementThe element the listener is attached to
event.preventDefault()methodStops the browser's default behaviour (form submit, link navigate)
event.stopPropagation()methodStops the event from bubbling up the DOM tree
event.keystringKey that was pressed: "Enter", "a", "ArrowUp"
event.codestringPhysical key code: "KeyA", "Space"
event.clientX / clientYnumberMouse position relative to viewport
event.shiftKey / ctrlKey / altKeybooleanWhether modifier keys were held
event.timeStampnumberWhen the event occurred (milliseconds)

event.target vs event.currentTarget

🎯

event.target — the element that was actually clicked/interacted with. This is where the event originated.

event.currentTarget — the element the addEventListener is attached to. These are different when events bubble: if you click a <span> inside a <button>, and the listener is on the button, then target is the <span> and currentTarget is the <button>.

preventDefault — Stop Default Browser Behaviour

Common uses of preventDefault()
// Stop a link from navigating
document.querySelector('a').addEventListener('click', (e) => {
  e.preventDefault();
  console.log('Link click intercepted — not navigating');
});

// Stop a form from reloading the page on submit
document.querySelector('form').addEventListener('submit', (e) => {
  e.preventDefault(); // ← always do this for SPA-style forms
  const data = new FormData(e.target);
  console.log('Form data:', Object.fromEntries(data));
});

// Stop right-click context menu
document.addEventListener('contextmenu', (e) => {
  e.preventDefault();
  // show custom context menu
});

Keyboard Events — event.key

app.js — handling key events
document.addEventListener('keydown', (e) => {
  console.log('Key:', e.key, '| Code:', e.code);

  // Check specific keys
  if (e.key === 'Enter') {
    console.log('Enter pressed!');
  }
  if (e.key === 'Escape') {
    console.log('Escape — closing modal');
  }
  // Modifier combinations
  if (e.ctrlKey && e.key === 's') {
    e.preventDefault(); // stop browser save dialog
    console.log('Ctrl+S — saving...');
  }
});

Try it — event object in action
Always name your event parameter: Convention is e or event. Inside the handler, e.target is almost always the first thing you'll access — it tells you exactly what the user interacted with.

Event Bubbling & Delegation

Events don't just fire on the clicked element — they travel up the DOM tree. Understanding this is what unlocks the most powerful event handling pattern: event delegation.

How Events Travel — Bubbling

When you click an element, the browser fires the click event on that element first, then on its parent, then its grandparent, all the way up to document. This is called bubbling.

HTML — nested elements
<div id="grandparent">
  <div id="parent">
    <button id="child">Click Me</button>
  </div>
</div>
app.js — click the button, all three fire
document.getElementById('grandparent').addEventListener('click', () => {
  console.log('3. grandparent — bubbled up here last');
});
document.getElementById('parent').addEventListener('click', () => {
  console.log('2. parent — bubbled up here second');
});
document.getElementById('child').addEventListener('click', () => {
  console.log('1. child — fired first (the element clicked)');
});

// Click the button → output:
// 1. child — fired first
// 2. parent — bubbled up here second
// 3. grandparent — bubbled up here last

Capturing vs Bubbling Phase

Bubbling (default)

Event travels up from target to root. This is the default phase and what you use 99% of the time. addEventListener('click', fn) or addEventListener('click', fn, false).

Capturing

Event travels down from root to target first. Use addEventListener('click', fn, true) or { capture: true }. Rarely needed — mainly for framework internals.

stopPropagation — Stop the Bubble

app.js
document.getElementById('parent').addEventListener('click', () => {
  console.log('parent listener');
});

document.getElementById('child').addEventListener('click', (e) => {
  e.stopPropagation(); // ← stops here, parent never fires
  console.log('child listener — bubble stopped');
});

// Click child → only "child listener" logs
// The parent's handler never runs
💡

Don't over-use stopPropagation. It can break other listeners (analytics, modals, etc.) that rely on events reaching document. Only stop propagation when you genuinely need to prevent a parent from reacting.

Event Delegation — The Best Pattern

Instead of adding a listener to every child element, add one listener to the parent. Check event.target to see which child was clicked. This is especially powerful for dynamic lists where items are added/removed.

Without delegation — adds N listeners
// ✗ Inefficient: 50 items = 50 listeners
document.querySelectorAll('.todo-item').forEach(item => {
  item.addEventListener('click', () => {
    item.classList.toggle('done');
  });
});
// Also breaks for items added AFTER this code runs!
With delegation — one listener handles everything
// ✓ Efficient: 1 listener for any number of items
const todoList = document.querySelector('#todo-list');

todoList.addEventListener('click', (e) => {
  // event.target is the element that was actually clicked
  const item = e.target.closest('.todo-item');
  if (!item) return; // clicked outside an item — do nothing

  item.classList.toggle('done');
  console.log('Toggled:', item.textContent);
});

// Now dynamically added items ALSO respond to clicks — for free!
function addItem(text) {
  const li = document.createElement('li');
  li.className = 'todo-item';
  li.textContent = text;
  todoList.appendChild(li);
}
addItem('New item — also clickable!'); // works without adding a listener

closest() — Navigate Up to Find a Matching Ancestor

app.js — event.target.closest()
// If a <li> contains: <span>Text</span><button class="delete">✕</button>
// Clicking the button gives e.target = the button
// But closest finds the parent li

list.addEventListener('click', (e) => {
  // Find the nearest ancestor (or self) matching the selector
  const item = e.target.closest('li');
  const deleteBtn = e.target.closest('.delete');

  if (deleteBtn) {
    item.remove();
  } else if (item) {
    item.classList.toggle('done');
  }
});

Try it — bubbling and delegation
Event delegation is the standard pattern for lists and any UI where items are added dynamically. One listener on the parent handles all children — current and future. Always use e.target.closest(selector) to safely find the intended target even when clicking a child element inside it.

Ask Yourself Before Starting

Q1

You need to handle both clicking an item (toggle done) and clicking a delete button. How do you use one listener for both?

Hint: Attach the listener to the parent <ul>. Inside, check e.target.closest('.delete-btn') — if truthy, delete the item; else toggle it. Use e.target.closest('li') to find the containing <li>.
Q2

How do you allow pressing Enter in the input to also add a task (not just clicking the button)?

Hint: Listen for keydown on the input. Check e.key === 'Enter'. Call the same addTask() function or trigger the button's click.
Q3

How do you show "No tasks yet" when the list is empty, and hide it when there are tasks?

Hint: Write a updateEmpty() function that checks list.children.length === 0 and toggles a CSS class or sets display on an empty-state element. Call it after adding or deleting.
