01

Arrays — Basics

An array is an ordered list of values. It is the most common data structure in JavaScript — used to hold collections of items like scores, names, products, or API results.

Creating an Array

Use square brackets [] to create an array. Items are separated by commas. Arrays can hold any type — including a mix of types — and they are zero-indexed: the first item is at index 0.

Creating arrays
// Array literal — always prefer this
const fruits   = ["apple", "banana", "mango"];
const scores   = [78, 85, 90, 62];
const mixed    = [42, "hello", true, null];

// Empty array — fill it later
const cart = [];

// Array constructor — rarely used
const blank = new Array(3);  // [empty × 3]

Accessing Elements

Index access — zero-based
const fruits = ["apple", "banana", "mango"];

console.log(fruits[0]);  // "apple"   ← first
console.log(fruits[1]);  // "banana"
console.log(fruits[2]);  // "mango"   ← last
console.log(fruits[3]);  // undefined ← out of bounds

// Access from the end
console.log(fruits[fruits.length - 1]);  // "mango"
console.log(fruits.at(-1));              // "mango" ← ES2022

Array Length

The .length property
const scores = [78, 85, 90, 62, 74];

console.log(scores.length);     // 5
console.log(scores.length - 1); // 4 — index of the last element

// length updates automatically as items are added/removed
scores.push(95);
console.log(scores.length);     // 6

Multidimensional Arrays

Arrays inside arrays
// 2D array — a grid / matrix
const matrix = [
  [1, 2, 3],
  [4, 5, 6],
  [7, 8, 9]
];

console.log(matrix[0]);     // [1, 2, 3]  ← row 0
console.log(matrix[1][2]);  // 6          ← row 1, column 2

// Practical: list of students with scores
const students = [
  ["Naman",  [85, 90, 78]],
  ["Priya",  [92, 88, 95]],
  ["Rahul",  [70, 75, 80]]
];

console.log(students[1][0]);     // "Priya"
console.log(students[1][1][2]);  // 95 ← Priya's third score
Mental model: Think of an array as a numbered shelf. Position 0 holds the first item, position 1 holds the second, and so on. fruits[0] means "reach into the shelf at slot 0". fruits.length tells you how many slots are filled.

Array Methods — Mutating

Mutating methods change the original array in place. You need to know which methods mutate and which don't — it prevents hard-to-find bugs.

Quick Reference — Mutating Methods

MethodWhat it doesExampleReturnsMDN
push(x) Add x to the end arr.push(4) New length MDN ↗
pop() Remove from the end arr.pop() Removed item MDN ↗
unshift(x) Add x to the start arr.unshift(0) New length MDN ↗
shift() Remove from the start arr.shift() Removed item MDN ↗
splice(i,n,...) Remove/insert at index i arr.splice(1,1,"x") Removed items MDN ↗
reverse() Reverse order in-place arr.reverse() The array MDN ↗
sort(fn) Sort in-place (default: lexicographic) arr.sort((a,b)=>a-b) The array MDN ↗
fill(v,s,e) Fill slots with value v arr.fill(0,0,3) The array MDN ↗

push / pop — End of Array

Stack behaviour — Last In, First Out
const stack = ["a", "b", "c"];

// push — add to end
stack.push("d");
console.log(stack);           // ["a", "b", "c", "d"]
console.log(stack.push("e")); // 5 ← returns new length

// pop — remove from end
const last = stack.pop();
console.log(last);  // "e"
console.log(stack); // ["a", "b", "c", "d"]

unshift / shift — Start of Array

Queue behaviour — First In, First Out
const queue = ["b", "c", "d"];

// unshift — add to front
queue.unshift("a");
console.log(queue);  // ["a", "b", "c", "d"]

// shift — remove from front
const first = queue.shift();
console.log(first);  // "a"
console.log(queue);  // ["b", "c", "d"]

splice — Remove, Insert, or Replace

splice(startIndex, deleteCount, ...itemsToInsert)
const arr = ["a", "b", "c", "d", "e"];

// Remove 2 items starting at index 1
const removed = arr.splice(1, 2);
console.log(removed); // ["b", "c"]
console.log(arr);     // ["a", "d", "e"]

// Insert without removing (deleteCount = 0)
arr.splice(1, 0, "x", "y");
console.log(arr);     // ["a", "x", "y", "d", "e"]

// Replace: remove 1, insert 1
arr.splice(2, 1, "Z");
console.log(arr);     // ["a", "x", "Z", "d", "e"]

sort — The Gotcha

⚠️

By default, sort() converts elements to strings and sorts lexicographically. This means [10, 9, 2].sort() gives [10, 2, 9] — not what you want for numbers. Always pass a comparator function for numeric sorts.

Correct numeric and string sort
// WRONG for numbers
[10, 9, 2, 100].sort();          // [10, 100, 2, 9] — lexicographic!

// CORRECT for numbers — ascending
[10, 9, 2, 100].sort((a, b) => a - b);  // [2, 9, 10, 100]

// CORRECT for numbers — descending
[10, 9, 2, 100].sort((a, b) => b - a);  // [100, 10, 9, 2]

// Strings (default sort works fine for ASCII)
["banana", "apple", "cherry"].sort();   // ["apple", "banana", "cherry"]

Try It — Mutating Methods

Try it
Push/pop vs unshift/shift: push/pop work at the end (fast — O(1)). unshift/shift work at the start (slower — O(n) because every element shifts). Prefer push/pop for performance-sensitive code.

Array Methods — Non-Mutating

Non-mutating methods return a new value and leave the original array unchanged. These are safer to use — you never accidentally modify the source data.

Quick Reference — Non-Mutating Methods

MethodWhat it doesExampleReturnsMDN
slice(s,e) Extract from index s up to (not including) e arr.slice(1,3) New array MDN ↗
concat(...) Merge arrays together a.concat(b, c) New array MDN ↗
includes(v) Check if array contains v arr.includes("a") Boolean MDN ↗
indexOf(v) Index of first occurrence of v arr.indexOf("b") Index or -1 MDN ↗
join(sep) Join all elements into a string arr.join(", ") String MDN ↗
flat(depth) Flatten nested arrays arr.flat() New array MDN ↗
toReversed() Reverse without mutating (ES2023) arr.toReversed() New array MDN ↗
toSorted(fn) Sort without mutating (ES2023) arr.toSorted((a,b)=>a-b) New array MDN ↗

slice — Extract a Portion

slice(startIndex, endIndex) — end is exclusive
const fruits = ["apple", "banana", "cherry", "date", "elderberry"];

// slice(1, 3) — from index 1 up to (not including) index 3
console.log(fruits.slice(1, 3));  // ["banana", "cherry"]

// slice(2) — from index 2 to the end
console.log(fruits.slice(2));     // ["cherry", "date", "elderberry"]

// Negative index — count from the end
console.log(fruits.slice(-2));    // ["date", "elderberry"]

// Shallow copy of entire array
const copy = fruits.slice();
console.log(copy === fruits);     // false — different reference

concat and flat

Merging and flattening
// concat — merge arrays
const a = [1, 2, 3];
const b = [4, 5, 6];
const merged = a.concat(b, [7, 8]);
console.log(merged);  // [1, 2, 3, 4, 5, 6, 7, 8]
console.log(a);       // [1, 2, 3] — unchanged

// Spread alternative (more common today)
const also = [...a, ...b];

// flat — collapse nested arrays
const nested = [1, [2, 3], [4, [5, 6]]];
console.log(nested.flat());      // [1, 2, 3, 4, [5, 6]] ← depth 1
console.log(nested.flat(2));     // [1, 2, 3, 4, 5, 6]   ← depth 2
console.log(nested.flat(Infinity)); // fully flatten any depth

join — Array to String

join is the reverse of split
const words = ["Hello", "World", "from", "JavaScript"];

console.log(words.join(" "));   // "Hello World from JavaScript"
console.log(words.join(", "));  // "Hello, World, from, JavaScript"
console.log(words.join(""));    // "HelloWorldfromJavaScript"
console.log(words.join("-"));   // "Hello-World-from-JavaScript"

// Common pattern: build a CSV row
const row = ["Naman", 85, "Pass"];
console.log(row.join(","));     // "Naman,85,Pass"

Try It — Non-Mutating Methods

Try it
slice vs splice: The names look similar — remember: slice (non-mutating, returns a portion) vs splice (mutating, changes the array). When in doubt, slice is the safe one.

Array Iteration

The iteration methods — map, filter, reduce, find, some, every, forEach — are among the most powerful tools in JavaScript. They replace most manual loops.

Quick Reference — Iteration Methods

MethodWhat it doesReturnsMDN
forEach(fn) Run fn for each item — like a for loop, no return value undefined MDN ↗
map(fn) Transform each item — returns a new array of the same length New array MDN ↗
filter(fn) Keep only items where fn returns true New array (shorter) MDN ↗
reduce(fn, init) Accumulate all items into one value (sum, object, string…) Single value MDN ↗
find(fn) Return first item where fn returns true Item or undefined MDN ↗
findIndex(fn) Return index of first match Index or -1 MDN ↗
some(fn) True if at least one item passes fn Boolean MDN ↗
every(fn) True only if ALL items pass fn Boolean MDN ↗

map — Transform Every Item

map returns a NEW array of the same length
const prices = [100, 250, 80, 400];

// Add 18% GST to every price
const withGST = prices.map(price => price * 1.18);
console.log(withGST);  // [118, 295, 94.4, 472]
console.log(prices);   // [100, 250, 80, 400] — unchanged!

// Extract a field from an array of objects
const students = [
  { name: "Naman",  score: 85 },
  { name: "Priya",  score: 92 },
  { name: "Rahul",  score: 70 }
];
const names = students.map(s => s.name);
console.log(names);  // ["Naman", "Priya", "Rahul"]

filter — Keep Matching Items

filter returns a NEW array — possibly shorter
const scores = [45, 78, 92, 55, 88, 34, 95];

// Keep only passing scores (>= 60)
const passing = scores.filter(score => score >= 60);
console.log(passing);  // [78, 92, 88, 95]

// Filter objects — students who passed
const students = [
  { name: "Naman", score: 85, passed: true  },
  { name: "Raj",   score: 42, passed: false },
  { name: "Priya", score: 91, passed: true  }
];
const passers = students.filter(s => s.passed);
console.log(passers.map(s => s.name)); // ["Naman", "Priya"]

reduce — Collapse to One Value

reduce(callback(accumulator, currentValue), initialValue)
const scores = [78, 85, 90, 62, 74];

// Sum all scores
const total = scores.reduce((acc, score) => acc + score, 0);
console.log("Total:", total);    // 389
console.log("Average:", total / scores.length);  // 77.8

// Max value using reduce
const max = scores.reduce((best, s) => s > best ? s : best, -Infinity);
console.log("Max:", max);  // 90

// Count occurrences — reduce to an object
const votes = ["yes", "no", "yes", "yes", "no"];
const tally = votes.reduce((acc, vote) => {
  acc[vote] = (acc[vote] || 0) + 1;
  return acc;
}, {});
console.log(tally);  // { yes: 3, no: 2 }

find, some, every

Search and check methods
const students = [
  { name: "Naman", score: 85 },
  { name: "Raj",   score: 42 },
  { name: "Priya", score: 91 }
];

// find — first match or undefined
const topStudent = students.find(s => s.score >= 90);
console.log(topStudent);  // { name: "Priya", score: 91 }

const missing = students.find(s => s.score > 100);
console.log(missing);     // undefined

// some — at least one passes?
console.log(students.some(s => s.score < 50));  // true (Raj)

// every — all pass?
console.log(students.every(s => s.score >= 60)); // false (Raj)
console.log(students.every(s => s.score > 0));   // true

Chaining Methods

Each method returns an array — chain them
const products = [
  { name: "Shirt",  price: 800,  inStock: true  },
  { name: "Jeans",  price: 1500, inStock: true  },
  { name: "Shoes",  price: 3000, inStock: false },
  { name: "Cap",    price: 400,  inStock: true  }
];

// Get names of in-stock items under ₹1000, sorted by price
const result = products
  .filter(p => p.inStock && p.price < 1000)
  .sort((a, b) => a.price - b.price)
  .map(p => `${p.name} (₹${p.price})`);

console.log(result);  // ["Cap (₹400)", "Shirt (₹800)"]

Try It — Iteration Methods

Try it
Choosing the right method: Need a new transformed array? → map. Need a subset? → filter. Need one value from all items? → reduce. Need to find one item? → find. Need a yes/no about items? → some or every. Just running side effects? → forEach.

Objects — Basics

An object is a collection of key–value pairs. Where an array uses numbered indexes, an object uses named keys. Objects model real-world entities — users, products, settings.

Creating an Object

Object literal — the standard way
// Object literal — key: value pairs
const student = {
  name:    "Naman",
  age:     20,
  course:  "Full Stack",
  passed:  true
};

// Empty object
const settings = {};

// Keys can be strings with spaces — use quotes
const weirdKeys = {
  "first name": "Naman",
  "user-id":    101
};

Accessing Properties

Dot notation vs bracket notation
const student = { name: "Naman", age: 20, course: "Full Stack" };

// Dot notation — clean, preferred when key is a known identifier
console.log(student.name);    // "Naman"
console.log(student.age);     // 20

// Bracket notation — required when key has spaces, or is dynamic
console.log(student["name"]); // "Naman"

const key = "course";
console.log(student[key]);    // "Full Stack" — dynamic lookup!

// Accessing a missing key
console.log(student.marks);   // undefined — no error

Adding, Updating, Deleting

Objects are mutable even when declared with const
const user = { name: "Naman", role: "student" };

// Add a new property
user.email = "naman@example.com";

// Update an existing property
user.role = "admin";

// Delete a property
delete user.email;

console.log(user); // { name: "Naman", role: "admin" }

// Check if a key exists
console.log("name" in user);   // true
console.log("email" in user);  // false

Nested Objects

Objects can contain objects (and arrays)
const student = {
  name: "Naman",
  address: {
    city:    "Delhi",
    pincode: "110001"
  },
  scores: [85, 90, 78, 92]
};

// Access nested properties
console.log(student.address.city);       // "Delhi"
console.log(student.address["pincode"]); // "110001"
console.log(student.scores[2]);          // 78

Methods — Functions Inside Objects

A method is an object property whose value is a function
const calculator = {
  result: 0,

  add(n) {
    this.result += n;
    return this;  // enables chaining
  },

  subtract(n) {
    this.result -= n;
    return this;
  },

  getValue() {
    return this.result;
  }
};

calculator.add(10).add(5).subtract(3);
console.log(calculator.getValue()); // 12

// this — refers to the object the method is called on
const greeter = {
  name: "Naman",
  greet() {
    console.log("Hello, I am " + this.name);
  }
};
greeter.greet(); // "Hello, I am Naman"

Try It — Objects

Try it
Dot vs bracket: Use dot notation (obj.key) when you know the key at write-time. Use bracket notation (obj[variable]) when the key is dynamic or stored in a variable. You'll need brackets for keys with spaces or special characters.

Object Methods

Object.keys(), Object.values(), and Object.entries() let you loop over, transform, and merge objects. Combined with spread, they cover most real-world object manipulation.

Quick Reference

MethodWhat it returnsExample outputMDN
Object.keys(obj) Array of own enumerable keys ["name", "age"] MDN ↗
Object.values(obj) Array of own enumerable values ["Naman", 20] MDN ↗
Object.entries(obj) Array of [key, value] pairs [["name","Naman"],["age",20]] MDN ↗
Object.assign(target, src) Copy src properties onto target Modified target MDN ↗
Object.fromEntries(pairs) Create object from [key,value] pairs { a: 1, b: 2 } MDN ↗
{ ...obj } Spread — shallow copy / merge New object MDN ↗

Object.keys / values / entries

Iterate over an object's data
const student = { name: "Naman", age: 20, score: 85 };

console.log(Object.keys(student));    // ["name", "age", "score"]
console.log(Object.values(student));  // ["Naman", 20, 85]
console.log(Object.entries(student)); // [["name","Naman"],["age",20],["score",85]]

// Loop over keys
for (const key of Object.keys(student)) {
  console.log(key, ":", student[key]);
}

// Loop over entries (destructured — see Ch 07)
for (const [key, value] of Object.entries(student)) {
  console.log(`${key} → ${value}`);
}

Copying and Merging with Spread

Spread is now preferred over Object.assign
const defaults = { theme: "light", lang: "en", fontSize: 16 };
const userPrefs = { theme: "dark", fontSize: 18 };

// Merge — userPrefs overrides defaults
const config = { ...defaults, ...userPrefs };
console.log(config);
// { theme: "dark", lang: "en", fontSize: 18 }

// Shallow copy — new object, same top-level values
const copy = { ...defaults };
copy.lang = "hi";
console.log(defaults.lang); // "en" — original unchanged
console.log(copy.lang);     // "hi"

// Add/override one field
const updated = { ...student, score: 95 };
console.log(updated); // same as student but score is 95

Object.fromEntries — Transform Object Values

entries → transform → fromEntries pattern
const prices = { shirt: 800, jeans: 1500, cap: 400 };

// Apply 10% discount to all prices
const discounted = Object.fromEntries(
  Object.entries(prices).map(([item, price]) => [item, price * 0.9])
);
console.log(discounted);
// { shirt: 720, jeans: 1350, cap: 360 }

Try It — Object Methods

Try it
Shallow vs deep copy: Spread ({ ...obj }) creates a shallow copy — nested objects are still shared by reference. If you need a truly independent deep copy, use structuredClone(obj) (modern browsers) or JSON.parse(JSON.stringify(obj)) (works everywhere but loses functions and dates).

07

Destructuring

Destructuring is a shorthand syntax to unpack values from arrays or properties from objects into named variables — making your code cleaner and more expressive.

Array Destructuring

Unpack by position
const scores = [85, 90, 78, 92];

// Old way
const first  = scores[0];
const second = scores[1];

// Destructuring — clean and concise
const [a, b, c, d] = scores;
console.log(a, b, c, d);  // 85 90 78 92

// Skip elements with commas
const [top, , third] = scores;
console.log(top, third);  // 85 78

// Rest — gather remaining items
const [head, ...rest] = scores;
console.log(head);  // 85
console.log(rest);  // [90, 78, 92]

// Default values
const [x = 0, y = 0, z = 0] = [10, 20];
console.log(x, y, z);  // 10 20 0

Object Destructuring

Unpack by key name
const student = { name: "Naman", age: 20, score: 85, course: "FullStack" };

// Extract by name
const { name, score } = student;
console.log(name, score);  // "Naman" 85

// Rename while destructuring
const { name: studentName, score: marks } = student;
console.log(studentName, marks);  // "Naman" 85

// Default values (if key doesn't exist)
const { name: n, passed = false } = student;
console.log(n, passed);  // "Naman" false

// Rest — gather remaining keys
const { name: nm, ...rest } = student;
console.log(nm);    // "Naman"
console.log(rest);  // { age: 20, score: 85, course: "FullStack" }

Nested Destructuring

Destructure deeply nested objects and arrays
const student = {
  name: "Naman",
  address: { city: "Delhi", pincode: "110001" },
  scores:  [85, 90, 78]
};

// Destructure nested object
const { name, address: { city, pincode } } = student;
console.log(name, city, pincode);  // "Naman" "Delhi" "110001"

// Destructure nested array
const { scores: [first, second] } = student;
console.log(first, second);  // 85 90

// Array of objects — destructure in a loop
const students = [
  { name: "Naman", score: 85 },
  { name: "Priya", score: 92 }
];
for (const { name, score } of students) {
  console.log(`${name}: ${score}`);
}

Destructuring in Function Parameters

Destructure directly in the parameter list
// Without destructuring
function greet(student) {
  console.log("Hello " + student.name + ", score: " + student.score);
}

// With destructuring — much cleaner
function greet({ name, score, course = "Unknown" }) {
  console.log(`Hello ${name}! Score: ${score}. Course: ${course}.`);
}

greet({ name: "Naman", score: 85, course: "FullStack" });
greet({ name: "Priya", score: 92 }); // course defaults to "Unknown"

Try It — Destructuring

Try it
Real-world use: Destructuring is everywhere in modern JS — React props, API response parsing, function parameters. The pattern const { data, error } = await fetch(...) is something you'll write daily. Learn it well.

Template Literals

Template literals use backticks instead of quotes. They allow you to embed expressions directly in strings and write multi-line text without hacks.

String Interpolation — ${}

Embed any expression directly in a string
const name  = "Naman";
const score = 85;
const pass  = score >= 60;

// Old way — concatenation
console.log("Hello " + name + ", your score is " + score);

// Template literal — clean
console.log(`Hello ${name}, your score is ${score}`);

// Any expression works inside ${}
console.log(`Pass: ${pass ? "Yes" : "No"}`);
console.log(`Double: ${score * 2}`);
console.log(`Grade: ${score >= 90 ? "A" : score >= 80 ? "B" : "C"}`);
console.log(`Uppercase: ${name.toUpperCase()}`);

Multi-Line Strings

No more \n — line breaks are literal
// Old way — backslash n
const old = "Line 1\nLine 2\nLine 3";

// Template literal — just press Enter
const modern = `Line 1
Line 2
Line 3`;

// Generating HTML (common React/DOM pattern)
const student = { name: "Naman", score: 85, course: "FullStack" };

const card = `
  <div class="student-card">
    <h2>${student.name}</h2>
    <p>Score: ${student.score}</p>
    <p>Course: ${student.course}</p>
  </div>
`;

Expressions Inside ${}

Function calls, ternaries, math — all valid
const items = ["Shirt", "Jeans", "Cap"];
const total = 2700;
const discount = 0.1;

// Function call inside template
console.log(`Items: ${items.join(", ")}`);

// Arithmetic
console.log(`Final price: Rs.${(total * (1 - discount)).toFixed(2)}`);

// Conditional
const stock = 0;
console.log(`Status: ${stock > 0 ? "In Stock" : "Out of Stock"}`);

// Nested template (template inside template)
const rows = items.map((item, i) => `${i + 1}. ${item}`).join("\n");
console.log(`Order summary:\n${rows}`);

Tagged Templates — Advanced

A function that processes a template literal
// A tag function receives the string parts and interpolated values
function highlight(strings, ...values) {
  return strings.reduce((result, str, i) => {
    const val = values[i] !== undefined ? `[${values[i]}]` : "";
    return result + str + val;
  }, "");
}

const name  = "Naman";
const score = 92;
console.log(highlight`Student ${name} scored ${score} marks.`);
// "Student [Naman] scored [92] marks."

// Real-world tagged templates: styled-components, GraphQL gql, sql

Try It — Template Literals

Try it
Backtick everywhere: Once you start using template literals, you'll rarely go back to quote concatenation. The rule is simple: if you're embedding a variable or expression into a string, use a template literal. For a plain string with no interpolation, either style is fine.

String Methods

Strings in JavaScript are immutable — every method returns a new string, never modifying the original. These methods cover the most common string operations you'll encounter daily.

Quick Reference — String Methods

MethodWhat it doesExampleReturnsMDN
.length Number of characters "hello".length → 5 Number MDN ↗
.toUpperCase() All characters uppercase "hi".toUpperCase() → "HI" String MDN ↗

Set

A Set is a collection of unique values — it automatically rejects duplicates. This makes it perfect for deduplication, membership checks, and tracking "seen" items.

Creating and Using a Set

new Set() — values are always unique
// Create from an array — duplicates are removed
const set = new Set([1, 2, 3, 2, 1, 4]);
console.log(set);       // Set(4) {1, 2, 3, 4}
console.log(set.size);  // 4 — not .length!

// Create empty, then add
const tags = new Set();
tags.add("javascript");
tags.add("arrays");
tags.add("javascript"); // duplicate — ignored
console.log(tags.size); // 2

The #1 Use Case — Deduplication

Remove duplicates from any array in one line
// Remove duplicate numbers
const nums = [1, 5, 3, 5, 2, 1, 4, 3];
const unique = [...new Set(nums)];
console.log(unique);  // [1, 5, 3, 2, 4]

// Remove duplicate strings
const tags = ["js", "html", "js", "css", "html", "js"];
const uniqueTags = [...new Set(tags)];
console.log(uniqueTags);  // ["js", "html", "css"]

// Count unique values
console.log(new Set(tags).size);  // 3

Set Operations

Union, intersection, and difference
const setA = new Set([1, 2, 3, 4]);
const setB = new Set([3, 4, 5, 6]);

// Union — all unique values from both
const union = new Set([...setA, ...setB]);
console.log([...union]);  // [1, 2, 3, 4, 5, 6]

// Intersection — only in both
const intersection = new Set([...setA].filter(x => setB.has(x)));
console.log([...intersection]);  // [3, 4]

// Difference — in A but not in B
const difference = new Set([...setA].filter(x => !setB.has(x)));
console.log([...difference]);  // [1, 2]

Try It — Set

Try it
Set vs Array: Use an Array when order matters and you need index access. Use a Set when you need uniqueness guaranteed, fast membership checks (has() is O(1) vs array's includes() which is O(n)), or deduplication. A Set has no index — to get a value by position, spread it to an array first.

Map

A Map is a collection of key–value pairs where keys can be any type — not just strings. Unlike plain objects, Maps maintain insertion order and have a built-in size property.

Creating and Using a Map

new Map() — keys can be any type
// Create empty, then set entries
const scores = new Map();
scores.set("Naman", 85);
scores.set("Priya", 92);
scores.set("Rahul", 70);

console.log(scores.get("Naman"));  // 85
console.log(scores.size);          // 3

// Create from an array of [key, value] pairs
const config = new Map([
  ["theme",    "dark"],
  ["lang",     "en"],
  ["fontSize", 16]
]);

// Keys can be any type — objects, numbers, functions
const objKey = { id: 1 };
const map = new Map();
map.set(objKey, "user data");
map.set(42, "numeric key");
console.log(map.get(objKey));  // "user data"
console.log(map.get(42));      // "numeric key"

Map vs Plain Object

Use Map when…

Keys are not strings (numbers, objects). You need to iterate in insertion order. You frequently add/remove keys. You need a quick .size. Data is stored as key-value but is not "an entity".

Use plain Object when…

Keys are strings or symbols. You're modelling an entity (user, product, config). You need JSON serialisation. You use property access syntax (obj.key).

Iterating a Map

Maps are iterable in insertion order
const scores = new Map([
  ["Naman", 85],
  ["Priya", 92],
  ["Rahul", 70]
]);

// for...of with destructuring
for (const [name, score] of scores) {
  console.log(`${name}: ${score}`);
}

// keys, values
console.log([...scores.keys()]);    // ["Naman", "Priya", "Rahul"]
console.log([...scores.values()]); // [85, 92, 70]

// Convert Map to plain object
const obj = Object.fromEntries(scores);
console.log(obj);  // { Naman: 85, Priya: 92, Rahul: 70 }

// Convert plain object to Map
const settings = { theme: "dark", lang: "en" };
const mapFromObj = new Map(Object.entries(settings));
console.log(mapFromObj.get("theme"));  // "dark"

Try It — Map

Try it
Map vs Object for dictionaries: Both work, but Map is faster for frequent insertions/deletions and automatically handles non-string keys. For static config objects ({ theme: "dark" }), use a plain object. For dynamic key-value stores that grow and shrink at runtime, use a Map.

grade-book.js — full solution
const students = [
  { id: 1, name: "Naman",  scores: [85, 90, 78] },
  { id: 2, name: "Priya",  scores: [92, 88, 95] },
  { id: 3, name: "Rahul",  scores: [55, 62, 48] },
  { id: 4, name: "Ayesha", scores: [77, 83, 79] },
  { id: 5, name: "Dev",    scores: [40, 35, 50]  }
];

// Step 1: Add avg using map + reduce
const graded = students.map(({ id, name, scores }) => {
  const avg = scores.reduce((a, b) => a + b, 0) / scores.length;
  return { id, name, scores, avg: +avg.toFixed(1) };
});

// Step 2: Print each student's report
console.log("=== Student Reports ===");
graded.forEach(({ id, name, scores, avg }) => {
  const grade = avg >= 90 ? "A" : avg >= 80 ? "B" : avg >= 70 ? "C" : avg >= 60 ? "D" : "F";
  const pass  = avg >= 60 ? "PASS" : "FAIL";
  console.log(
    `[${String(id).padStart(2,"0")}] ${name.padEnd(8)} ` +
    `| Scores: ${scores.join(" / ")} ` +
    `| Avg: ${avg.toFixed(1)} ` +
    `| ${grade} — ${pass}`
  );
});

// Step 3: Filter passing and failing
const passing = graded.filter(s => s.avg >= 60);
const failing = graded.filter(s => s.avg <  60);

// Step 4: Class average via reduce
const classAvg = graded.reduce((sum, s) => sum + s.avg, 0) / graded.length;

// Step 5: Top student
const top = graded.reduce((best, s) => s.avg > best.avg ? s : best, graded[0]);

// Summary
console.log(`
=== Class Summary ===
Total students : ${students.length}
Passed         : ${passing.length} (${passing.map(s => s.name).join(", ")})
Failed         : ${failing.length} (${failing.map(s => s.name).join(", ")})
Class average  : ${classAvg.toFixed(1)}
Top student    : ${top.name} (${top.avg})
====================`);
