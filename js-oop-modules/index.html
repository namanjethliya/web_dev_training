01

this in Different Contexts

this is one of JavaScript's most misunderstood keywords. Its value is not fixed — it changes depending on how a function is called, not where it is defined.

The Golden Rule

🔑

this refers to the object that is currently calling the function. If no object is calling it, this falls back to the global object (window in browsers) or undefined in strict mode.

Context 1: Global Scope

At the top level of a script (outside any function or object), this is the global object — window in the browser.

global scope
console.log(this);          // Window { ... }
console.log(this === window); // true (in browser)

Context 2: Regular Functions

Inside a regular function called without an object, this is window (or undefined in strict mode).

regular function
function greet() {
  console.log(this); // Window — no object called this function
}
greet();

// In strict mode:
"use strict";
function strictGreet() {
  console.log(this); // undefined — strict mode prevents global fallback
}
strictGreet();

Context 3: Object Methods

When a function is called as a method of an object, this is that object.

object method
const user = {
  name: "Naman",
  greet() {
    console.log("Hello, I am", this.name); // 'this' is the user object
  }
};

user.greet(); // Hello, I am Naman

// DANGER: if you detach the method, 'this' is lost
const fn = user.greet;
fn(); // Hello, I am undefined — 'this' is now window, not user!
⚠️

The most common this bug: passing an object method as a callback. When the callback is eventually called, the original object context is lost. This is why .bind() exists — covered in Chapter 03.

Context 4: Event Handlers

In a DOM event handler, this refers to the element that received the event.

event handler
// HTML: <button id="btn">Click me</button>
const btn = document.getElementById("btn");

btn.addEventListener("click", function() {
  console.log(this);       // <button id="btn"> — the element
  console.log(this.id);    // "btn"
});

// Arrow function version — 'this' is NOT the element (see Ch. 02)
btn.addEventListener("click", () => {
  console.log(this); // Window — arrow functions don't bind 'this'
});

Try It — this in Object Methods

Try it
Rule of thumb: Ask yourself "what is to the left of the dot when this function is called?" That object is this. If there is nothing to the left of the dot, this is either window or undefined (strict mode).

02

Arrow Functions & this

Arrow functions do not have their own this. Instead, they capture this from the surrounding scope at the time they are defined — this is called lexical this.

The Difference: Regular vs Arrow

Regular function — own this

Creates its own this each time it is called. The value depends on how the function is called.

const obj = {
  name: "Naman",
  greet: function() {
    console.log(this.name); // "Naman"
  }
};
obj.greet(); // works
Arrow function — lexical this

Inherits this from the enclosing scope. Does not create its own this.

const obj = {
  name: "Naman",
  greet: () => {
    console.log(this.name); // undefined
    // 'this' is window here, not obj
  }
};
obj.greet(); // broken

Where Arrow Functions Shine: Nested Callbacks

Before arrow functions, developers had to save this to a variable (often called self or that) to use it inside nested functions. Arrow functions solve this elegantly.

The problem arrow functions solved
// OLD WAY — saving 'this' as a workaround
function Timer() {
  const self = this; // save 'this' before losing it
  self.seconds = 0;
  setInterval(function() {
    self.seconds++; // use 'self' instead of 'this'
    console.log(self.seconds);
  }, 1000);
}

// MODERN WAY — arrow function inherits 'this' from Timer
function Timer() {
  this.seconds = 0;
  setInterval(() => {
    this.seconds++; // 'this' is still the Timer instance — lexical!
    console.log(this.seconds);
  }, 1000);
}

Practical Rule: When to Use Each

SituationUseWhy
Object method that needs to access this function or method shorthand Needs its own this bound to the object
Class method Method shorthand (no keyword) Same — needs to bind to the instance
Callback inside a method (.map, setTimeout) Arrow function Inherits this from the outer method — no binding needed
Standalone utility function Either No this needed — use arrow for brevity
Event handler where this = element function Arrow would capture outer this, losing the element reference

Try It — Lexical this

Try it
Memory trick: Arrow functions are arrows — they point back to where they were born. Their this is always the this of the place they were defined, not the place they are called.

03

bind, call, apply

These three methods let you explicitly control what this is inside a function — instead of letting JavaScript decide based on how the function is called.

The Problem They Solve

losing this
const user = {
  name: "Naman",
  greet() {
    console.log("Hello, I am", this.name);
  }
};

// Works fine
user.greet(); // "Hello, I am Naman"

// Breaks — 'this' is lost when the method is assigned to a variable
const sayHello = user.greet;
sayHello(); // "Hello, I am undefined"

// Fix with bind:
const boundHello = user.greet.bind(user);
boundHello(); // "Hello, I am Naman" ✓

call() — Call immediately with a specific this

Function.prototype.call(thisArg, arg1, arg2, ...)
function introduce(greeting, punctuation) {
  console.log(`${greeting}, I am ${this.name}${punctuation}`);
}

const person1 = { name: "Naman" };
const person2 = { name: "Raj" };

introduce.call(person1, "Hello", "!");   // "Hello, I am Naman!"
introduce.call(person2, "Hi", ".");      // "Hi, I am Raj."

// call() invokes the function immediately
// Arguments are passed individually after thisArg

apply() — Like call(), but args are an array

Function.prototype.apply(thisArg, [arg1, arg2, ...])
function introduce(greeting, punctuation) {
  console.log(`${greeting}, I am ${this.name}${punctuation}`);
}

const person1 = { name: "Naman" };
const args = ["Hello", "!"];

introduce.apply(person1, args); // "Hello, I am Naman!"
// apply() is useful when args are already in an array

// Classic use: Math.max with an array
const numbers = [3, 1, 4, 1, 5, 9, 2, 6];
console.log(Math.max.apply(null, numbers)); // 9
// Modern equivalent: Math.max(...numbers)

bind() — Returns a new function with this fixed

Function.prototype.bind(thisArg, arg1, arg2, ...)
function introduce(greeting, punctuation) {
  console.log(`${greeting}, I am ${this.name}${punctuation}`);
}

const person1 = { name: "Naman" };

// bind returns a NEW function — does not call it immediately
const boundIntroduce = introduce.bind(person1, "Hello");
boundIntroduce("!");  // "Hello, I am Naman!"
boundIntroduce("?");  // "Hello, I am Naman?"

// Common use: preserve 'this' in event listeners / callbacks
const timer = {
  count: 0,
  start() {
    // Without .bind(this), 'this' would be lost inside setInterval
    const tick = function() {
      this.count++;
      console.log("Count:", this.count);
    }.bind(this); // fixes 'this' to the timer object

    // In practice — just use an arrow function instead (Ch. 02)
    const interval = setInterval(tick, 100);
    setTimeout(() => clearInterval(interval), 400);
  }
};
timer.start();

Quick Comparison

MethodCalls the function?Args formatReturnsUse when
call() Yes — immediately Comma-separated Function's return value You know the args and want to run now
apply() Yes — immediately Array Function's return value Args are in an array (spread operator is modern replacement)
bind() No — returns new fn Comma-separated (partial) New bound function You want to save a function for later with this fixed

Try It — bind, call, apply

Try it
Modern practice: In class-based code, use arrow functions for callbacks to avoid this loss — it's cleaner than .bind(this). Use call()/apply() when borrowing methods from one object to use on another.

Classes — Basics

Classes are blueprints for creating objects. They group related data (properties) and behaviour (methods) together, making code easier to organise, reuse, and maintain.

Defining a Class

Syntax
class Person {
  // constructor runs once when you create a new instance
  constructor(name, age) {
    this.name = name; // instance property
    this.age  = age;
  }

  // Method — shared by all instances (not copied per object)
  greet() {
    console.log(`Hi, I'm ${this.name} and I'm ${this.age} years old.`);
  }

  isAdult() {
    return this.age >= 18;
  }
}

// Creating instances with 'new'
const person1 = new Person("Naman", 25);
const person2 = new Person("Priya", 17);

person1.greet();               // Hi, I'm Naman and I'm 25 years old.
person2.greet();               // Hi, I'm Priya and I'm 17 years old.
console.log(person1.isAdult()); // true
console.log(person2.isAdult()); // false

What 'new' Does — Step by Step

Step 1

Creates a new empty object: {}

Step 2

Sets the object's prototype to the class's prototype (so methods are shared)

Step 3

Runs the constructor with this pointing to the new object

Step 4

Returns the new object (unless you explicitly return another object)

Adding More Methods

BankAccount class
class BankAccount {
  constructor(owner, initialBalance = 0) {
    this.owner   = owner;
    this.balance = initialBalance;
  }

  deposit(amount) {
    if (amount <= 0) throw new Error("Deposit must be positive");
    this.balance += amount;
    console.log(`Deposited ₹${amount}. New balance: ₹${this.balance}`);
  }

  withdraw(amount) {
    if (amount > this.balance) throw new Error("Insufficient funds");
    this.balance -= amount;
    console.log(`Withdrew ₹${amount}. New balance: ₹${this.balance}`);
  }

  getStatement() {
    return `Account: ${this.owner} | Balance: ₹${this.balance}`;
  }
}

const acc = new BankAccount("Naman", 1000);
acc.deposit(500);   // Deposited ₹500. New balance: ₹1500
acc.withdraw(200);  // Withdrew ₹200. New balance: ₹1300
console.log(acc.getStatement()); // Account: Naman | Balance: ₹1300

Try It — Classes

Try it
Class vs Object Literal: Use an object literal ({ }) for a single, one-off thing. Use a class when you need to create multiple similar objects — classes are reusable blueprints. If you need 50 User objects, define a User class, not 50 object literals.

05

Inheritance

Inheritance lets a class build on another class — sharing its properties and methods while adding or changing behaviour. In JavaScript, use the extends keyword.

extends and super()

Base class + subclass
// Base class (parent / superclass)
class Animal {
  constructor(name, sound) {
    this.name  = name;
    this.sound = sound;
  }

  speak() {
    console.log(`${this.name} says: ${this.sound}!`);
  }

  describe() {
    return `I am ${this.name}, an animal.`;
  }
}

// Subclass (child) — extends inherits everything from Animal
class Dog extends Animal {
  constructor(name, breed) {
    // super() calls the parent constructor — MUST be called first
    super(name, "Woof");
    this.breed = breed; // add a new property specific to Dog
  }

  // Add a new method that only Dog has
  fetch(item) {
    console.log(`${this.name} fetches the ${item}!`);
  }

  // Override the inherited describe() method
  describe() {
    return `I am ${this.name}, a ${this.breed} dog.`;
  }
}

const dog = new Dog("Bruno", "Labrador");
dog.speak();           // Bruno says: Woof!  — inherited from Animal
dog.fetch("ball");     // Bruno fetches the ball! — Dog-specific
console.log(dog.describe()); // I am Bruno, a Labrador dog. — overridden

The super Keyword

super() in constructor

Calls the parent class's constructor. Must be the first line in a subclass constructor — calling this before super() throws a ReferenceError.

super.method() in methods

Calls a method from the parent class. Use it when you want to extend a parent method rather than completely replace it.

describe() {
  // call parent version first, then add to it
  const base = super.describe();
  return base + ` (${this.breed})`;
}

instanceof — Checking the Type

instanceof
class Animal { }
class Dog extends Animal { }
class Cat extends Animal { }

const dog = new Dog();

console.log(dog instanceof Dog);    // true
console.log(dog instanceof Animal); // true — Dog IS an Animal
console.log(dog instanceof Cat);    // false

// instanceof checks up the prototype chain
console.log(dog instanceof Object); // true — everything is an Object

Multi-level Inheritance

Chaining classes
class Vehicle {
  constructor(make, speed) {
    this.make  = make;
    this.speed = speed;
  }
  move() { console.log(`${this.make} moving at ${this.speed} km/h`); }
}

class Car extends Vehicle {
  constructor(make, speed, doors) {
    super(make, speed);
    this.doors = doors;
  }
  honk() { console.log(`${this.make}: Beep beep!`); }
}

class ElectricCar extends Car {
  constructor(make, speed, doors, range) {
    super(make, speed, doors);
    this.range = range;
  }
  charge() { console.log(`${this.make} charging... range: ${this.range} km`); }
}

const tesla = new ElectricCar("Tesla", 250, 4, 500);
tesla.move();   // inherited from Vehicle
tesla.honk();   // inherited from Car
tesla.charge(); // ElectricCar-specific

Try It — Inheritance

Try it
When to inherit: Use inheritance for an "is-a" relationship — a Dog IS an Animal, a Manager IS an Employee. For "has-a" relationships (a Car HAS an Engine), use composition instead — store the object as a property.

Getters, Setters & Static Methods

Getters and setters let you compute or validate values when properties are accessed or assigned. Static methods belong to the class itself, not to instances.

Getters — Computed Properties

A getter looks like a property when accessed but runs a function. Use get before the method name.

get keyword
class Temperature {
  constructor(celsius) {
    this._celsius = celsius; // convention: _ prefix = "private-ish"
  }

  // Getter — accessed like a property, not a function call
  get fahrenheit() {
    return (this._celsius * 9) / 5 + 32;
  }

  get description() {
    if (this._celsius < 0)  return "Freezing";
    if (this._celsius < 15) return "Cold";
    if (this._celsius < 25) return "Mild";
    return "Warm";
  }
}

const temp = new Temperature(30);
console.log(temp.fahrenheit);   // 86 — no () needed!
console.log(temp.description);  // "Warm"
console.log(temp._celsius);     // 30 (accessible but _ signals "private")

Setters — Validated Assignment

A setter runs when a property is assigned. Use it to validate or transform values before storing them.

set keyword
class Circle {
  constructor(radius) {
    this.radius = radius; // uses the setter!
  }

  get radius() {
    return this._radius;
  }

  set radius(value) {
    if (value < 0) throw new Error("Radius cannot be negative");
    this._radius = value;
  }

  get area() {
    return Math.PI * this._radius ** 2;
  }

  get circumference() {
    return 2 * Math.PI * this._radius;
  }
}

const c = new Circle(5);
console.log(c.radius);            // 5
console.log(c.area.toFixed(2));   // 78.54

c.radius = 10;                    // uses the setter
console.log(c.area.toFixed(2));   // 314.16

try {
  c.radius = -1;                  // setter throws
} catch (e) {
  console.log("Error:", e.message);
}

Static Methods — Class-Level Utilities

Static methods are called on the class itself, not on instances. They don't have access to this (the instance) — they're utilities related to the class.

static keyword
class MathUtils {
  // static — called on the class, not an instance
  static square(n) { return n * n; }
  static cube(n)   { return n * n * n; }
  static clamp(value, min, max) {
    return Math.min(Math.max(value, min), max);
  }
}

// Call on the CLASS — no 'new' needed
console.log(MathUtils.square(5));       // 25
console.log(MathUtils.cube(3));         // 27
console.log(MathUtils.clamp(15, 0, 10)); // 10

// Real example: factory method pattern
class User {
  constructor(name, role) {
    this.name = name;
    this.role = role;
  }

  static createAdmin(name) {
    return new User(name, "admin"); // static factory
  }

  static createGuest() {
    return new User("Guest", "guest");
  }
}

const admin = User.createAdmin("Naman");  // cleaner than new User("Naman", "admin")
const guest = User.createGuest();
console.log(admin.name, admin.role); // Naman admin
console.log(guest.name, guest.role); // Guest guest

Try It — Getters, Setters, Static

Try it
When to use each: Use getters for derived/computed values (area, fullName, totalPrice). Use setters for validation before storing a value. Use static methods for factory methods, utility functions, and things that don't depend on instance state.

Higher-Order Functions

A higher-order function either accepts a function as an argument or returns a function (or both). They are the foundation of functional programming patterns in JavaScript.

Functions as Arguments (Callbacks)

You have already used higher-order functions without knowing — forEach, map, filter, and sort all accept functions as arguments.

Functions as arguments
// A simple higher-order function
function doTwice(fn) {
  fn();
  fn();
}

function sayHi() { console.log("Hi!"); }

doTwice(sayHi);    // Hi!
                   // Hi!

// The real power: the caller decides the behaviour
function transform(array, fn) {
  const result = [];
  for (const item of array) {
    result.push(fn(item));
  }
  return result;
}

const numbers = [1, 2, 3, 4, 5];

const doubled  = transform(numbers, n => n * 2);   // [2, 4, 6, 8, 10]
const squared  = transform(numbers, n => n ** 2);   // [1, 4, 9, 16, 25]
const asString = transform(numbers, n => `#${n}`);  // ["#1","#2","#3","#4","#5"]

console.log(doubled);
console.log(squared);
console.log(asString);
// Note: this is exactly what Array.prototype.map does!

Functions Returning Functions — Closures

A function that returns another function creates a closure — the returned function remembers the variables of its outer function even after it has returned.

Function factories
// A function that creates specialised functions
function multiplier(factor) {
  return function(number) {
    return number * factor; // 'factor' is remembered via closure
  };
}

const double = multiplier(2);
const triple = multiplier(3);
const tenX   = multiplier(10);

console.log(double(5));  // 10
console.log(triple(5)); // 15
console.log(tenX(5));   // 50

// Real-world use: creating validators
function makeMinLength(min) {
  return function(str) {
    return str.length >= min;
  };
}

const isValidPassword = makeMinLength(8);
const isValidUsername = makeMinLength(3);

console.log(isValidPassword("abc"));      // false
console.log(isValidPassword("secret123")); // true
console.log(isValidUsername("Jo"));       // false
console.log(isValidUsername("Naman"));    // true

Function Composition

Composition means combining small, focused functions to build more complex behaviour. The output of one function becomes the input of the next.

compose and pipe
// Small, focused functions
const trim     = str => str.trim();
const toLower  = str => str.toLowerCase();
const removeSpaces = str => str.replace(/\s+/g, "-");

// Manual composition (right-to-left like math: g(f(x)))
const toSlug = str => removeSpaces(toLower(trim(str)));
console.log(toSlug("  Hello World  ")); // "hello-world"

// pipe — compose left-to-right (easier to read)
function pipe(...fns) {
  return function(value) {
    return fns.reduce((v, fn) => fn(v), value);
  };
}

const makeSlug = pipe(trim, toLower, removeSpaces);
console.log(makeSlug("  Hello World  ")); // "hello-world"
console.log(makeSlug("  JavaScript OOP  ")); // "javascript-oop"

// Compose multiple transforms with pipe
const processName = pipe(
  str => str.trim(),
  str => str.replace(/\s+/g, " "),         // collapse spaces
  str => str.split(" ").map(w =>           // title-case
    w[0].toUpperCase() + w.slice(1).toLowerCase()
  ).join(" ")
);

console.log(processName("  john   doe  ")); // "John Doe"

Practical Patterns

Memoisation

Cache expensive computation results.

function memoize(fn) {
  const cache = {};
  return function(n) {
    if (cache[n] !== undefined) return cache[n];
    cache[n] = fn(n);
    return cache[n];
  };
}
Currying

Transform a multi-arg function into a chain of single-arg functions.

const add = a => b => a + b;
const add5 = add(5);
console.log(add5(3)); // 8
console.log(add5(7)); // 12
Once

Run a function only once, no matter how many times called.

function once(fn) {
  let called = false;
  return function(...args) {
    if (!called) {
      called = true;
      return fn(...args);
    }
  };
}

Try it
Why higher-order functions matter: They let you write flexible, reusable code. Instead of hard-coding behaviour, you pass it in as a function. Array.map, Array.filter, Array.reduce — every one of them is a higher-order function you use every day.

08

ES6 Modules

Modules let you split your code across multiple files and control what each file exposes to the rest of the application. They make large codebases maintainable.

ℹ️

Note on this chapter: ES6 modules require a server or bundler to run — they can't be executed in this sandbox with eval(). This chapter shows you the syntax and explains each piece. The sandbox at the end simulates module concepts using plain functions so you can see the ideas in action.

Why Modules?

Without modules

All code in one giant file. Variables from one part accidentally overwrite another. Impossible to maintain beyond a few hundred lines.

With modules

Each file is its own scope. You explicitly choose what to share (export) and what to import. Dependencies are clear. Files are focused and testable.

Named Exports

Export multiple named things from one file. The consumer imports them by name.

mathUtils.js — exporting
// Named exports — export individual items
export function add(a, b)      { return a + b; }
export function subtract(a, b) { return a - b; }
export function multiply(a, b) { return a * b; }

export const PI = 3.14159265;

// Or export at the bottom (same result)
function divide(a, b) { return a / b; }
export { divide };
app.js — importing named exports
// Import by name — must match the exported name exactly
import { add, multiply, PI } from './mathUtils.js';

console.log(add(2, 3));         // 5
console.log(multiply(4, 5));    // 20
console.log(PI);                // 3.14159265

// Rename on import with 'as'
import { subtract as minus } from './mathUtils.js';
console.log(minus(10, 3));      // 7

// Import everything into a namespace object
import * as MathUtils from './mathUtils.js';
console.log(MathUtils.add(1, 2)); // 3
console.log(MathUtils.PI);        // 3.14159265

Default Export

Each module can have one default export. It can be imported with any name.

User.js — default export
// Default export — only one per file
export default class User {
  constructor(name, email) {
    this.name  = name;
    this.email = email;
  }
  greet() {
    return `Hello, I am ${this.name}`;
  }
}
app.js — importing the default export
// No curly braces for default export — choose any name
import User from './User.js';
import MyUser from './User.js'; // also valid — same export, different local name

const u = new User("Naman", "naman@example.com");
console.log(u.greet()); // Hello, I am Naman

Mixing Named and Default

api.js
// One default + multiple named exports in the same file
export default class ApiClient { /* ... */ }

export const BASE_URL = "https://api.example.com";
export function formatResponse(data) { return JSON.stringify(data); }
app.js
// Default first, then named in braces
import ApiClient, { BASE_URL, formatResponse } from './api.js';

Re-exporting — Barrel Files

A common pattern is to create an index.js that re-exports from many files, giving consumers a single import point.

utils/index.js — barrel file
// Re-export from child files
export { add, subtract }  from './mathUtils.js';
export { formatDate }     from './dateUtils.js';
export { default as User } from './User.js';
app.js — clean single import
// One import covers everything
import { add, formatDate, User } from './utils/index.js';

Using Modules in the Browser

index.html
<!-- type="module" enables ES6 module support -->
<!-- Modules are automatically deferred (load after HTML) -->
<script type="module" src="app.js"></script>

Prototype & Prototype Chain

JavaScript's class system is built on prototypes under the hood. Understanding this explains why instanceof works, how methods are shared efficiently, and what the prototype chain lookup actually does.

Every Object Has a Prototype

Prototype basics
const dog = { name: "Bruno", breed: "Labrador" };

// Every object has a hidden [[Prototype]] link
// Access it with Object.getPrototypeOf()
const proto = Object.getPrototypeOf(dog);
console.log(proto === Object.prototype); // true

// dog itself doesn't have 'toString', but the prototype chain does
console.log(dog.toString()); // "[object Object]" — found on Object.prototype

// Checking own properties vs inherited
console.log(dog.hasOwnProperty("name"));     // true — dog's own property
console.log(dog.hasOwnProperty("toString")); // false — inherited

How Classes Use Prototypes

class is syntactic sugar over prototype-based inheritance. Methods defined in a class are stored on the class's prototype object — not copied into each instance. This saves memory.

Under the hood
class Animal {
  constructor(name) { this.name = name; }
  speak() { console.log(`${this.name} speaks`); }
}

const a = new Animal("Leo");

// 'name' lives on the instance (a.name)
// 'speak' lives on Animal.prototype — shared!
console.log(a.hasOwnProperty("name"));  // true  — own property
console.log(a.hasOwnProperty("speak")); // false — on prototype

// Proof: both instances share the same speak function
const b = new Animal("Rex");
console.log(a.speak === b.speak); // true — same function reference!

The Prototype Chain

When you access a property, JavaScript searches the object first, then its prototype, then that prototype's prototype, and so on — until it finds it or reaches null.

Chain lookup
class Animal {
  constructor(name) { this.name = name; }
  breathe() { return `${this.name} breathes`; }
}

class Dog extends Animal {
  bark() { return `${this.name} barks`; }
}

const dog = new Dog("Bruno");

// Property lookup order:
// 1. dog (own properties) — name? YES → "Bruno"
// 2. Dog.prototype        — bark? YES → function
// 3. Animal.prototype     — breathe? YES → function
// 4. Object.prototype     — toString? YES → function
// 5. null                 — (end of chain)

console.log(dog.name);     // "Bruno" — own property
console.log(dog.bark());   // "Bruno barks" — Dog.prototype
console.log(dog.breathe()); // "Bruno breathes" — Animal.prototype
console.log(dog.toString()); // "[object Object]" — Object.prototype

// Visualising the chain
console.log(Object.getPrototypeOf(dog) === Dog.prototype);    // true
console.log(Object.getPrototypeOf(Dog.prototype) === Animal.prototype); // true
console.log(Object.getPrototypeOf(Animal.prototype) === Object.prototype); // true
console.log(Object.getPrototypeOf(Object.prototype)); // null — end of chain

prototype vs __proto__

PropertyBelongs toPoints toUsage
Constructor.prototype Functions/classes The prototype object for instances Where methods live. Dog.prototype.bark
Object.getPrototypeOf(obj) All objects The object's prototype Standard, safe way to access prototype
obj.__proto__ All objects Same as getPrototypeOf Legacy/non-standard — avoid, use getPrototypeOf instead

Adding to Prototypes

Extending built-in prototypes — with caution
// You can add methods to any prototype (including built-ins)
// — but modifying built-ins is generally BAD practice
// — it can conflict with library code or future JS features

// OK: adding to your own class's prototype after definition
class Greeter {
  constructor(name) { this.name = name; }
}
Greeter.prototype.sayHi = function() {
  return `Hi from ${this.name}`;
};
const g = new Greeter("Naman");
console.log(g.sayHi()); // "Hi from Naman"

// BAD practice — don't do this in real code:
// String.prototype.shout = function() { return this.toUpperCase() + "!!!"; };
// "hello".shout(); // "HELLO!!!" — works but pollutes built-ins

class Shape {
  constructor(name) {
    this.name = name;
  }

  get area() {
    throw new Error("area must be implemented by subclass");
  }

  get perimeter() {
    throw new Error("perimeter must be implemented by subclass");
  }

  describe() {
    return `${this.name}: area=${this.area.toFixed(2)}, perimeter=${this.perimeter.toFixed(2)}`;
  }

  static compare(s1, s2) {
    if (s1.area > s2.area) return `${s1.name} is larger (${s1.area.toFixed(2)} vs ${s2.area.toFixed(2)})`;
    if (s2.area > s1.area) return `${s2.name} is larger (${s2.area.toFixed(2)} vs ${s1.area.toFixed(2)})`;
    return "Both shapes have equal area";
  }
}

class Circle extends Shape {
  constructor(radius) {
    super("Circle");
    this.radius = radius; // uses setter
  }

  get radius()      { return this._radius; }
  set radius(value) {
    if (value < 0) throw new Error("Radius cannot be negative");
    this._radius = value;
  }

  get area()      { return Math.PI * this._radius ** 2; }
  get perimeter() { return 2 * Math.PI * this._radius; }

  describe() {
    return `Circle (r=${this._radius}): area=${this.area.toFixed(2)}, perimeter=${this.perimeter.toFixed(2)}`;
  }
}

class Rectangle extends Shape {
  constructor(width, height) {
    super("Rectangle");
    if (width < 0 || height < 0) throw new Error("Dimensions cannot be negative");
    this.width  = width;
    this.height = height;
  }

  get area()      { return this.width * this.height; }
  get perimeter() { return 2 * (this.width + this.height); }

  describe() {
    return `Rectangle (${this.width}×${this.height}): area=${this.area.toFixed(2)}, perimeter=${this.perimeter.toFixed(2)}`;
  }
}
